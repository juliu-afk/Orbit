```
POST /api/v1/goal {defer_to_offpeak:true, target_provider:"deepseek", ...}
  │
  ├─ 1. CreateGoalRequest → GoalSession (含 4 新字段)
  │
  ├─ 2. [分支] if defer_to_offpeak and not urgent:
  │     │
  │     ├─ 2a. PreFlightEstimator.estimate(description)
  │     │      → PreFlightResult(token_low, token_high, time_low, time_high)
  │     │
  │     ├─ 2b. OffPeakScheduler.enqueue(goal, estimate)
  │     │      ├─ PeakWindowManager.is_peak(provider) → bool
  │     │      ├─ PeakWindowManager.next_offpeak_window(provider) → PeakWindow
  │     │      ├─ estimate_window_capacity(window, queued) → int
  │     │      ├─ DeferredQueue.push(DeferredTask) → SQLite INSERT
  │     │      └─ return {status:"queued", target_window_start, ...}
  │     │
  │     └─ 2c. 返回 {"goal_id":..., "status":"queued", ...}
  │
  └─ 3. [else] orch.run(goal) — 现有路径，不变

后台协程:
  OffPeakScheduler._window_watcher()  # 每分钟检查
    ├─ 窗口即将开始(≤5min)?
    │   └─ DeferredQueue.pop_for_window(window) → List[DeferredTask]
    │       └─ 逐任务: orch.run(goal) — 同现有调用
    │
    ├─ 窗口即将结束(≤10min)?
    │   └─ 检查剩余容量 → 不够则标记顺延
    │
    └─ 窗口已结束?
        └─ 停止释放新任务，运行中任务继续跑完
```

**判定算法**:

```python
def is_peak(self, provider: str, at: datetime | None = None) -> bool:
    config = self._config[provider]
    now = at or datetime.now(ZoneInfo(config.timezone))
    day_name = now.strftime("%a")  # Mon, Tue, ...
    time_str = now.strftime("%H:%M")

    for window in config.peak_windows:
        if day_name in window.days:
            if window.hours_start <= time_str < window.hours_end:
                return True
    return False
```

**节假日处理**: 从 `ORBIT_HOLIDAYS_URL` 获取节假日 JSON（格式: `{"2026-07-04": "US Independence Day", ...}`），若当前日期是节假日，则高峰期判定返回 `False`（全天视为低峰）。

### 3.3 OffPeakScheduler

**文件**: `src/orbit/scheduler/offpeak_scheduler.py`

```python
class OffPeakScheduler:
    """高峰避让调度器——协调 PeakWindowManager + DeferredQueue + MetaOrchestrator。

    WHY 独立类: 遵循单一职责——PeakWindowManager 管时间判定，
    DeferredQueue 管持久化队列，OffPeakScheduler 管协调。

    Usage:
        offpeak = OffPeakScheduler(
            peak_manager=PeakWindowManager(),
            queue=DeferredQueue("data/offpeak.db"),
            orchestrator=meta_orch,     # 引用——窗口到了调 orch.run()
            preflight=PreFlightEstimator(),
        )
        await offpeak.start()  # 启动 _window_watcher 后台协程
    """

    def __init__(
        self,
        peak_manager: PeakWindowManager,
        queue: DeferredQueue,
        orchestrator: MetaOrchestrator,
        preflight: PreFlightEstimator,
    ):
        self._peak = peak_manager
        self._queue = queue
        self._orch = orchestrator
        self._preflight = preflight
        self._watcher_task: asyncio.Task | None = None
        self._force_offpeak_only = os.getenv("ORBIT_OFFPEAK_ONLY", "") == "true"

    async def enqueue(self, goal: GoalSession) -> EnqueueResult:
        """将 Goal 排队到低峰窗口。

        1. 调 PreFlightEstimator.estimate() 获取预估
        2. 判定当前是否高峰 → 确定目标窗口
        3. 构造 DeferredTask → push 到队列
        """
        # 1. 预估
        estimate = await self._preflight.estimate(goal.description)
        avg_tokens = (estimate.token_low + estimate.token_high) // 2
        avg_duration = (estimate.time_low_seconds + estimate.time_high_seconds) // 2

        # 2. 确定目标窗口
        provider = goal.target_provider or "deepseek"  # 默认 DeepSeek
        if self._peak.is_peak(provider):
            window = self._peak.next_offpeak_window(provider)
        else:
            # 当前已是低峰，立即执行（但不走 orch.run——仍排队，下一轮 watcher 释放）
            window = self._peak._current_window(provider)

        # 3. 构造任务并入队
        task = DeferredTask(
            id=goal.id,
            goal_description=goal.description,
            priority=goal.priority if hasattr(goal, 'priority') else "NORMAL",
            provider=provider,
            estimated_tokens=avg_tokens,
            estimated_duration_seconds=avg_duration,
            target_window_start=window.starts_at_iso,
            target_window_end=window.ends_at_iso,
            status="queued",
            created_at=datetime.now(UTC).isoformat(),
            goal_json=goal.model_dump_json(),
        )
        position = self._queue.push(task)
        return EnqueueResult(
            goal_id=goal.id,
            target_window_start=window.starts_at_iso,
            target_window_end=window.ends_at_iso,
            queue_position=position,
        )

    async def start(self) -> None:
        """启动后台窗口监视协程。"""
        self._watcher_task = asyncio.create_task(self._window_watcher())

    async def _window_watcher(self) -> None:
        """后台协程：每分钟检查是否有低峰窗口即将开始。

        WHY 每分钟: 在精度（不错过窗口）和资源（不空转）间平衡。
        """
        while True:
            try:
                await self._check_windows()
            except Exception:
                logger.exception("window_watcher_error")
            await asyncio.sleep(60)

    async def _check_windows(self) -> None:
        """检查所有活跃厂商的低峰窗口。"""
        for provider in self._peak.providers:
            window = self._peak.next_offpeak_window(provider)
            if window is None:
                continue

            now = datetime.now(UTC)
            window_start = datetime.fromisoformat(window.starts_at_iso)
            window_end = datetime.fromisoformat(window.ends_at_iso)
            minutes_to_start = (window_start - now).total_seconds() / 60

            # 窗口前 5 分钟: 预热（检查 API 可用性）
            if 0 < minutes_to_start <= 5:
                logger.info("window_preheating", provider=provider, starts_at=window.starts_at_iso)
                # 预热逻辑: ping 一次 LLM API 确认可用

            # 窗口已开始: 释放排队任务
            if minutes_to_start <= 0 and now < window_end:
                # 窗口还剩多少容量
                capacity = estimate_window_capacity(
                    window_start, window_end,
                    self._queue.list_all(status_filter="queued"),
                )
                if capacity > 0:
                    tasks = self._queue.pop_for_window(
                        window.starts_at_iso, window.ends_at_iso, limit=capacity
                    )
                    for task in tasks:
                        goal = GoalSession.model_validate_json(task.goal_json)
                        self._queue.mark_released(task.id)
                        asyncio.create_task(self._orch.run(goal))

            # 窗口结束前 10 分钟: 检查剩余容量，不够则顺延
            minutes_to_end = (window_end - now).total_seconds() / 60
            if 0 < minutes_to_end <= 10:
                remaining = self._queue.list_all(status_filter="queued")
                capacity = estimate_window_capacity(now, window_end, remaining)
                if len(remaining) > capacity:
                    # 标记超出容量的任务顺延到下一个窗口
                    next_window = self._peak.next_offpeak_window(provider, after=window_end)
                    overflow = remaining[capacity:]
                    for task in overflow:
                        # 更新 target_window 到下一个窗口
                        self._queue.reschedule(task.id, next_window)
                    logger.warning(
                        "window_overflow",
                        provider=provider,
                        overflow_count=len(overflow),
                        next_window=next_window.starts_at_iso,
                    )
```

### 3.4 窗口容量算法

```python
def estimate_window_capacity(
    window_start: datetime,
    window_end: datetime,
    queued_tasks: list[DeferredTask],
    max_parallel: int = 5,
) -> int:
    """估算低峰窗口能跑几个任务。

    WHY 悲观估算: 用预估耗时的 1.3x 做 buffer。
    WHY 短任务优先: 最大化吞吐——先跑小任务填满窗口。
    """
    window_seconds = (window_end - window_start).total_seconds()
    sorted_tasks = sorted(
        queued_tasks,
        key=lambda t: (
            # 优先级权重: CRITICAL/HIGH 排前面
            0 if t.priority in ("CRITICAL", "HIGH") else 1,
            t.estimated_duration_seconds,  # 耗时短的优先
        ),
    )

    remaining = window_seconds * max_parallel
    count = 0
    for task in sorted_tasks:
        cost = task.estimated_duration_seconds * 1.3  # 30% buffer
        if remaining >= cost:
            remaining -= cost
            count += 1
        else:
            break
    return count
```

### 3.5 成本计算

```python
# gateway/routing.py 扩展
def select_model(
    strategy: RoutingStrategy,
    available_models: list[str] | None = None,
    price_multiplier: float = 1.0,  # NEW: 当前时段价格倍数
) -> RoutingDecision:
    """选择模型——price_multiplier 调整有效成本。"""
    if strategy == RoutingStrategy.CHEAPEST:
        # NEW: 用有效成本排序（基础成本 × 时段倍数）
```

## 6. 调度器影响分析

### 6.1 状态变更

无新增调度器状态。`OffPeakScheduler` 是外层包装，不改变 `MetaOrchestrator` 内部的 `GoalSession.status` 状态机。

`DeferredTask.status` 是独立的状态机：

```
queued → released → running → done
  │         │                    ↑
  │         └─ (超时/失败) → queued（顺延到下一窗口）
  ├─ urgent_override → 直接释放
  └─ cancelled
```

### 6.2 检查点策略

不涉及检查点变更。`OffPeakScheduler` 的 `DeferredQueue` 持久化在 `deferred_tasks` 表，本身就是持久状态，无需额外检查点。

### 6.3 并发考虑

- 单例 `OffPeakScheduler` —— `_window_watcher` 只有一个协程
- `DeferredQueue` 使用 `asyncio.Lock` 保护 SQLite 写操作（遵循 `LoopScheduler` 的 `asyncio.to_thread` 模式）
- 并行释放的多个任务通过 `asyncio.create_task(orch.run())` 并发执行，由 `ResourceScheduler` 管理并发上限

## 9. 边界 Case 清单

| # | 场景 | 预期行为 |
|---|------|---------|
| B1 | 提交 `defer_to_offpeak=true` 但 OffPeakScheduler 未初始化 | 返回 503 "OffPeakScheduler 未初始化" |
| B2 | 当前已是低峰 + 提交延迟任务 | 目标窗口设为当前窗口，下一轮 watcher 立即释放 |
| B3 | 低峰窗口容量不够放下所有排队任务 | 按优先级+短任务优先排序，超出的顺延到下一个窗口 |
| B4 | 低峰窗口到期但仍有任务在运行 | 不强制中断——运行中任务跑完，新任务停止释放 |
| B5 | 服务重启 + `deferred_tasks` 表有排队任务 | `DeferredQueue.__init__` 加载所有 `queued` 任务，watcher 恢复后继续调度 |
| B6 | `urgent=true` + `defer_to_offpeak=true` 同时设置 | `urgent` 优先——忽略 defer，直接 `orch.run()` |
| B7 | `target_provider` 为空 | 默认 `"deepseek"` |
| B8 | 用户 `urgent` 提升已在执行中的任务 | 返回 409 "任务已在执行" |
| B9 | PreFlightEstimator 预估严重偏低（实际耗时远超预估） | 不阻断队列，运行任务超时不强制 kill，记录 `estimation_error` 指标 |
| B10 | 节假日判定——配置文件未覆盖某国假日 | 若 `ORBIT_HOLIDAYS_URL` 配置了 JSON API，从 API 获取；否则仅按 YAML 配置 |
| B11 | YAML 配置文件格式错误 | 启动时 `PeakWindowManager.__init__` 校验，格式错误抛 `ValueError` + 用内置默认值兜底 |
| B12 | `ORBIT_OFFPEAK_ONLY=true` + 用户提交 `urgent=true` | `urgent` 例外——允许立即执行 |

## 10. 风险与缓解

| # | 风险 | 等级 | 缓解 |
|---|------|------|------|
| R1 | `PreFlightEstimator` 预估偏差大 → 窗口容量计算不准 → 任务溢出 | 中 | 30% buffer + 溢出顺延 + `estimation_error` 指标持续校准 |
| R2 | watcher 检查频率（每分钟）可能错过极短窗口 | 低 | 窗口 <10 分钟无实际意义——LLM 调用至少 30s；可配置检查间隔 |
| R3 | `goal_json` 序列化/反序列化 GoalSession 可能丢失字段 | 中 | 单元测试覆盖所有 GoalSession 字段；Pydantic `model_validate_json` 有完整校验 |
| R4 | SQLite WAL 模式 + 多协程并发写 → 潜在锁竞争 | 低 | `asyncio.Lock` 保护所有写操作；`DeferredQueue` 使用 `asyncio.to_thread` 异步化同步 SQLite 操作 |
| R5 | 厂商分时定价变化后 YAML 配置过时 | 低 | 支持手动 `reload-config` API + 文档提醒定期更新 |
