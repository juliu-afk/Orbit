1. 系统总览

Orbit = 轻量级多Agent软件开发自循环系统。定位在编排层——不是单Agent执行工具，而是让一群AI协作完成软件开发全流程的治理系统。

核心价值: 编排 治理 验证 追溯

ProcessGuard 4项检查 (每次状态转换前执行)

检查1: 强制状态不可跳过
PARSING 和 CODING 是硬性强制状态。LLM 试图跳过→ ProcessViolationError 阻断。
判定方式: 比较 _STATE_ORDER，若已越过强制状态但未 visited→违规。
检查2: PLANNING 跳过权限
CODING 前 PLANNING 未 visited 且未授权快车道→违规。必须 ComplexityScorer 授权。
检查3: VERIFYING 跳过权限
DONE 前 VERIFYING 未 visited 且未授权快车道→违规。
检查4: 前一状态产物完整性
前驱状态的 artifact 必须存在（如 PARSING→PLANNING 时 PARSING 必须有产物）。

管道执行策略

validate_quick()

快速验证 L1+L4+L3
用于每次代码生成后即时检查
不需要 Sandbox 的层同步运行

validate_full()

全量验证 L1-L8
用于 CODING 阶段完成后的完整验证
依赖缺失的层自动跳过

fail-open 原则

单个验证器崩溃不阻塞管道
记录 warning → 继续下一层
仅 L1/L7 失败立即停止
