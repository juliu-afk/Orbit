## 数据流

```
文件变更 → tree-sitter 多语言解析 → CodeNode/Edge (不变schema)
                                     ↓
                          query_graph(type, filters) ← 统一入口
                                     ↓
                          Agent 查询 / 防幻觉 L1 校验
                          
git diff → change_detector → 受影响符号 + 风险 → 调度器任务失效
                 ↓
          graph.db.zst ← artifact.py (zstd 压缩/解压)
```

### 3. query_graph() 统一入口（graph/query.py）

```
async def query_graph(
    type: Literal["code","database","config"],
    symbol: str = None,
    table: str = None,
    key: str = None,
    **filters,
) -> dict:
    """设计文档 §10.4——统一图谱查询入口"""
    if type == "code":
        return await code_graph.query(symbol, **filters)
    elif type == "database":
        return await db_graph.query(table, **filters)
    elif type == "config":
        return await config_graph.query(key, **filters)
```

## 风险点

| 风险 | 缓解 |
|------|------|
| tree-sitter grammar 版本不兼容 | 锁定 grammar 版本到 tree-sitter 官方 release tag |
| 多语言索引时间增加 | Python 项目仍<30s（设计文档SC1）。TS前端<500文件秒级 |
| zstd 导入后本地diff过大→增量索引慢 | 仅对新增/修改文件增量——不解压全量重建 |
| 现有测试回归 | `_parse_file` 签名不变——Python文件仍走同逻辑。CI门禁 |
