## 3. API 设计

本批次无新增 REST API 端点。验证器以 Python 类接口暴露：

```python
# L1
class L1GraphValidator:
    def __init__(self, code_engine: CodeGraphEngine)
    async def validate(self, code: str) -> ValidationResult

# L2
class L2DynamicTracer:
    def __init__(self, sandbox: Sandbox)
    async def validate(self, code: str) -> ValidationResult

# L3
class L3EntropyMonitor:
    def __init__(self, config: L3EntropyConfig)
    def on_token(self, token: str, logprobs: list[float] | None) -> float | None
    def should_cancel(self) -> bool

# L4
class L4TypeValidator:
    def __init__(self, mypy_path: str = "mypy")
    async def validate(self, code: str) -> ValidationResult
```

## 4. 数据模型

```python
# src/orbit/hallucination/schemas.py

class HallucinationLevel(str, Enum):
    L1_GRAPH = "l1_graph"
    L2_DYNAMIC = "l2_dynamic"
    L3_ENTROPY = "l3_entropy"
    L4_TYPE = "l4_type"

class ValidationResult(BaseModel):
    passed: bool
    level: HallucinationLevel
    errors: list[str] = []
    warnings: list[str] = []
    metadata: dict[str, Any] = {}

class L3EntropyConfig(BaseModel):
    window_size: int = 10
    threshold: float = 0.75
    fallback_enabled: bool = True  # 无 logprobs 时降级为重复度检测
```

异常类：
```python
class HallucinationError(Exception): ...
class GraphReferenceError(HallucinationError): ...  # L1
class HighEntropyError(HallucinationError): ...     # L3
class TypeCheckError(HallucinationError): ...       # L4
```

## 5. 数据流

```
生成的代码片段
    │
    ├──→ L1GraphValidator.validate(code)
    │       └── ast.parse(code) → 提取 Name(id) → CodeGraphEngine.exists(name)
    │           └── 任一不存在 → ValidationResult(passed=False)
    │
    ├──→ L4TypeValidator.validate(code)
    │       └── 写临时文件 → subprocess: mypy --strict temp.py
    │           └── returncode != 0 → 解析错误行 → ValidationResult(passed=False)
    │
    ├──→ L2DynamicTracer.validate(code)
    │       └── 注入 trace 包装代码 → Sandbox.run(wrapped_code)
    │           └── 收集 traced_calls → 对照 CodeGraphEngine.exists()
    │
    └──→ L3EntropyMonitor（生成过程中逐 token 调用）
            └── on_token(token, logprobs) → 滑动窗口均值 ≥ threshold
                └── should_cancel() → True → 抛出 HighEntropyError
```

## 7. 边界 case 清单

| 场景 | 预期行为 | 覆盖测试 |
|------|---------|---------|
| 代码为空字符串 | L1/L4 返回 passed=True（无符号可验证） | `test_l1_empty_code` / `test_l4_empty_code` |
| LLM 模型不支持 logprobs | L3 降级为文本重复度检测，记录 warning | `test_l3_fallback_repetition` |
| mypy 未安装或执行失败 | L4 返回 passed=False，errors 含具体原因 | `test_l4_mypy_not_found` |
| 代码图谱查询超时 | L1 返回 passed=False，errors=["Graph query timeout"] | `test_l1_graph_timeout` |
| 高熵窗口未满（<10 采样） | L3 不触发，should_cancel() 返回 False | `test_l3_window_not_full` |
| 沙箱不可用 | L2 跳过，返回 passed=True + warnings | `test_l2_sandbox_unavailable` |
| AST 解析失败（语法错误） | L1 返回 passed=False，errors 含 SyntaxError 详情 | `test_l1_syntax_error` |

## 8. 风险与缓解

| # | 风险 | 严重度 | 缓解 |
|----|------|--------|------|
| R1 | L2 sys.settrace 在 Docker 容器内行为不确定（Python 版本差异） | 中 | 注入代码含 Python 版本检测；CI 用 Python 3.12 固定版本测试 |
| R2 | L4 mypy subprocess 开销大（每次调用 ~500ms 冷启动） | 低 | 仅对生成代码运行，非热路径；后续可换 mypy API 调用 |
| R3 | L3 当前无法从 LLMClient 获取 streaming logprobs（_do_completion 非流式） | 中 | **本批次 L3 仅交付独立监控器（纯函数 + 降级策略），不集成到 LLMClient 流式管道。** 流式集成留待 Step 5.1（调度器 DAG）统一改造 LLMClient。测试用模拟 token 序列，不依赖真实 LLM |
| R4 | L1 对内置函数（print/len）报假阳性 | 低 | AST 提取时跳过 builtins 模块符号（`__builtins__`），仅验证用户定义符号 |
