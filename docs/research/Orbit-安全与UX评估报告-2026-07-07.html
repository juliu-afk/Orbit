执行摘要

安全：Orbit 拥有完善的安全架构骨架（Docker 沙箱 + ProcessSandbox + 权限引擎 + 熔断器），但存在 3 个 P0 致命漏洞。最严重的是 WebSocket 聊天端点无认证，以及 .env 文件含真实 API 密钥未入 .gitignore。对标业界（Claude Code/Cursor/GitHub Copilot）的沙箱化标准，Orbit 的 Windows 沙箱隔离不完整，缺少 Firecracker/gVisor 级别的硬件隔离。


UX：Orbit 的终端风格设计和键盘导航是亮点，但缺少模式选择（Ask/Edit/Agent）、计划透明性、Diff 可视化、@-mention 上下文注入、持久化记忆系统等关键交互模式。与 Cursor、Copilot Chat、Windsurf 等竞品的 UX 差距约 2 年。14 项可执行改进中，前 4 项（模式选择器、Diff 可视化、Agent 计划展示、@-mention）可带来最大用户体验提升。

1.2 致命发现 (P0)

P0-1: WebSocket 聊天端点无认证 — src/orbit/api/routes/chat.py:89
AuthMiddleware 仅拦截 HTTP 请求，WebSocket 升级请求绕过中间件。任何能访问 18888 端口的人可无限制调用 LLM Agent。修复：在 WebSocket accept 前验证 token。
P0-2: .env 文件含真实 API 密钥且未入 .gitignore — d:/Orbit/.env
DEEPSEEK_API_KEY、ZAI_API_KEY 为真实密钥格式。若推送公开仓库即泄露。修复：立即将 .env 加入 .gitignore，轮换已暴露的密钥。
P0-3: Shell 工具使用 create_subprocess_shell — src/orbit/tools/shell.py:264
尽管有命令白名单和 shlex.split，shell=True 模式仍存在 shell 解释风险。修复：改用 create_subprocess_exec(list, ...)，直接 exec 无需 shell 中介。

2.3 关键 UX 差距

Gap 1: 无交互模式选择 — 用户无法区分"提问"vs"编辑"vs"自主执行"。所有输入走同一 NL 通道。
方案：在 TerminalChat 输入框上方添加模式选择器 (Ask / Edit / Agent / Dream)，借鉴 Copilot 和 Cursor 的三模式设计。工作量：低，影响：高。
Gap 2: 无 Diff 可视化 — CodeDiffPanel 是原始 <pre> 文本，非真正的 diff。
方案：集成 Monaco DiffEditor，添加 Accept/Reject 按钮，借鉴 Cursor 的 Diff & Review。工作量：中，影响：高。
Gap 3: Agent 执行无计划预览 — ChatStream 直接开始执行，用户不知道 Agent 要做什么。
方案：ChatStream 启动前展示计划卡片（步骤列表 + Approve/Skip/Modify），借鉴 Claude Code 的 plan-first loop。工作量：中，影响：高。
Gap 4: 无 @-mention 上下文注入 — 不能引用文件、符号、历史。
方案：在 InputBox 实现 `@` 自动完成（文件/符号/会话），高亮 @mention 为样式化 chip。工作量：中，影响：高。
Gap 5: 无持久化记忆/规则系统 — DreamPanel 是一次性 API 调用。无项目级配置。
方案：添加 Rules 面板（类似 Windsurf AGENTS.md / Copilot instructions），添加 Memory 浏览器。工作量：中，影响：高。
