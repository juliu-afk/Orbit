## 实施优先级

```

Harness: read/write/edit/exec/grep/glob 工具    (3天)
Loop: ReAct 循环 + tool calling                 (2天)

```

## 各 Agent 差异

| Agent | 需要文件工具 | 需要 Shell | 需要搜索 | 记忆需求 |
|-------|:--:|:--:|:--:|:--:|
| architect | ✅ read/grep/glob | ❌ | ✅ | 方案记忆 |
| developer | ✅ read/write/edit | ✅ exec | ✅ grep/glob | 代码记忆 |
| reviewer | ✅ read/grep | ❌ | ✅ | 审查模式 |
| qa | ✅ read | ✅ exec | ❌ | 测试用例 |
| config_manager | ✅ read/write | ❌ | ❌ | 配置快照 |
| clarifier | ❌ | ❌ | ❌ | 对话历史 |

> 文件工具不是所有 Agent 都需要——按角色授权，减少安全面。

## 关键决策

1. **先做 Harness (工具) 再做 Prompt**——没有工具的 Agent 改不了代码，prompt 再好也没用
2. **记忆系统先做最简单的**——对标 OpenClaw 的双文件（MEMORY.md + 日志），不要一上来就知识图谱
3. **失败升级直接对接已有 escalation.py**——不需要新架构
4. **安全权限边做边加**——先白名单再沙箱，不要一开始就建完整的 7 层权限系统
