# Orbit 操作手册

> 版本 0.12 | 适用于 Windows 11

---

## 1. 安装与启动

### 1.1 环境要求
- Windows 11（推荐），Windows 10 64位
- 8GB 以上内存
- 需要网络连接（调用 AI 模型）
- 可选：Git（用于代码管理）

### 1.2 启动步骤

1. 下载 `Orbit.exe`（约 39MB）
2. 双击运行，等待 5-10 秒启动
3. 浏览器自动打开 Dashboard 页面
4. 看到连接状态显示"已连接"即启动成功

> 注意：首次启动可能需要 30 秒加载 AI 模型配置。

## 3. 日常操作

### 3.1 创建项目

1. 点击顶栏项目区 → 弹出对话框
2. 输入项目名称（如"我的记账软件"）
3. 选一个代码模板（可选）
4. 点击"创建"
5. Orbit 会初始化项目目录和 Git 仓库

### 3.2 日常编程——聊天模式

在聊天框里直接说话，例如：

> "帮我在 src/utils 下写一个计算两数之和的函数"

Orbit 会：
1. 分析你的需求
2. 读取项目结构
3. 生成代码
4. 展示给你审核

你可以继续对话：
- "可以，但改成 TypeScript" → 修改
- "再写个测试" → 追加测试
- "不太对，我想要..." → 纠正

### 3.3 大任务——Compose 模式

适合"一次性想清楚需求，让 AI 自动执行"的场景。

1. 点击右侧面板的 **「Spec Compose」**
2. 在弹出的窗口粘贴任务说明书（YAML 格式）：

```yaml
title: 实现用户登录功能
tasks:
  - id: backend-login
    description: 实现用户名密码登录的后端 API
    agent_role: developer
  - id: frontend-login
    description: 实现登录页面的前端表单
    agent_role: developer
    depends_on: [backend-login]
  - id: review-login
    description: 审查登录代码的安全性
    agent_role: reviewer
    depends_on: [backend-login, frontend-login]
```

3. 点击"编排执行"
4. Orbit 会自动：
   - 检查方案是否合理
   - 按顺序执行各子任务（有依赖的先做）
   - 代码写完后自动审查
   - 显示每个任务的执行结果

### 3.4 记忆优化

Orbit 会自动记住项目经验。你也可以手动触发优化：

1. 点击右侧面板的 **「记忆优化」**
2. Orbit 会执行 5 阶段记忆合并：
   - 收集最近的记忆
   - 去重合并相似内容
   - 提炼关键洞察
   - 验证结果
   - 存储优化后的记忆

优化后，下次聊天时 Orbit 会更精准地引用过去的经验。

## 4. 查看系统状态

### 4.1 基础指标（右侧面板）

| 指标 | 含义 |
|------|------|
| 活跃任务 | 当前正在执行的 Agent 任务数 |
| Token 消耗 | AI 模型的调用量（每次对话都会消耗 Token） |
| 防幻觉拦截 | AI 输出中被拦截的错误/虚假信息次数 |
| 熔断器 | 系统保护机制——超限时自动暂停 |
| 系统健康 | 综合评分：正常 / 降级 / 异常 |

### 4.2 详情面板（点击"详情 ▸"）

- **Token 折线图**：展示最近 Token 消耗曲线
- **告警列表**：活跃的系统告警
- **组件健康**：各子系统的健康状态

## 5. 权限与安全

Orbit 内置权限控制，防止 AI 执行危险操作：

| 操作 | 权限 |
|------|------|
| 读取文件 | ✅ 所有 Agent |
| 搜索代码 | ✅ 所有 Agent |
| 写文件 | ✅ 仅 Developer、QA |
| 执行命令 | ⚠️ 仅 Developer、QA，需经过沙箱隔离 |
| 修改配置文件 | ❌ Config Manager 专用 |
| 访问密钥 | ❌ 全部禁止 |

## 6. 常见问题

**Q: AI 写了错误的代码怎么办？**
A: 直接告诉它"不对，应该..."。Orbit 会从错误中学习，记忆系统也会降低错误方案的权重。

**Q: 我的代码会被上传到云端吗？**
A: 不会。代码和记忆都在你的本地电脑上。AI 调用是通过加密传输的提示词（不包含完整项目代码）。

**Q: 可以离线使用吗？**
A: 需要网络连接调用 AI 模型。代码编辑、测试执行等本地操作无需网络。

**Q: 如何升级？**
A: 下载新版本 Orbit.exe，替换旧文件即可。记忆和项目数据不受影响。

## 7. 术语表

| 术语 | 含义 |
|------|------|
| Agent | 一个 AI 工作单元，有特定角色和权限 |
| ReAct | 思考→行动→观察 循环，Agent 的工作方式 |
| Compose | 多 Agent 编排执行模式 |
| /dream | 记忆合并优化引擎 |
| Token | AI 模型处理的文本单位（≈ 0.75 个英文词） |
| 防幻觉 | 多层验证机制，防止 AI 输出虚假信息 |
| 熔断器 | 资源保护机制，超限时自动暂停 |
