🤖 Agent 连接微信 · 完整方案报告

基于 2026 年官方 iLink Bot API 及备选方案 · 含长消息拆分策略

让 AI Agent 接入微信，目前主要有两条技术路径：官方 iLink Bot API（推荐，稳定合规）与第三方桥接框架（备用，有风险）。本报告将聚焦官方方案的操作细节，并解答长消息（超过 2048 字符）的拆分处理。

✅ 方案一：官方 iLink Bot API（推荐）

官方 腾讯通过 OpenClaw 平台开放了个人微信的 Bot API，基于 iLink（智联）协议（接入域 ilinkai.weixin.qq.com）。该方案合规、稳定，是当前首选。

🛠 主流桥接工具

  • cc-weixin – 极简桥接器，专为 Claude Code 设计，一行命令启动。
  • weixin-agent-sdk – 通用 SDK，可对接任意 Agent（Python/Node.js）。
  • weixin-agent-rs – Rust 实现，跨平台支持良好。
  • Hermes Agent – 全功能 AI 助理框架，内置微信接入模块。
  • ZLAgent – 支持 Docker 一键部署，开箱即用。

📋 快速上手步骤（以 cc-weixin 为例）

  1. 环境准备：安装 Node.js ≥ 22，并确保 Agent 服务（如 Claude Code）或 API Key 可用。
  2. 启动桥接：在终端执行
    npx cc-weixin
  3. 扫码登录：终端会生成二维码，使用 微信小号 扫描并确认。登录凭证自动保存，下次无需重复扫码。
  4. 测试连通：向“文件传输助手”发送消息，Agent 应能正常响应。

💡 其他工具类似，通常运行 hermes gateway setup 或 python -m weixin_agent 即可。

⚠️ 方案二：第三方桥接框架（备用）

代表项目：wechat-bridge、itchat 等，通过模拟客户端收发消息。

风险提示：

  • 违反微信用户协议，存在 封号风险（尤其主号）。
  • 依赖非公开协议，稳定性差，可能随时失效。

仅建议在测试环境或对合规性要求不高的场景下考虑。

🗺️ 通用接入流程（两种方案皆适用）

  1. 服务器/环境：准备一台可稳定运行的服务器（如腾讯云轻量应用服务器），安装 Node.js 或 Python 环境。
  2. 选择 Agent 框架：根据需求选用 Hermes、ZLAgent 等，或自行封装 API。
  3. 配置微信接入：
    • 官方方案：运行桥接工具扫码登录。
    • 第三方方案：部署 wechat-bridge 等服务，配置消息转发。
  4. 测试与优化：发送测试消息，观察响应速度与准确性，针对长消息做拆分处理。

✂️ 关于 2048 字符限制与消息拆分

微信单条消息有字节数限制（约 680 个汉字 / 2048 字节）。Agent 的回复经常超过此长度，必须拆分成多条发送。

📌 如何处理？

  • 自动拆分：大多数 SDK（如 weixin-agent-sdk）已内置拆分逻辑，会根据字节边界自动切分并逐条发送。
  • 手动拆分：如需自行控制，可调用 bot.sendMessage() 或底层接口，按 Unicode 字符 边界循环发送。
  • 注意事项：
    • 避免在中文或 Emoji 中间截断，否则会显示乱码。
    • 建议每条消息控制在 600 汉字 以内，确保兼容。
    • 部分框架支持“合并转发”或“分段标记”，但普通文本逐条发送最为可靠。

💡 代码示例（伪代码）

// 假设 agentReply 是 Agent 生成的完整回复 const MAX_BYTES = 2000; const chunks = splitByBytes(agentReply, MAX_BYTES); for (const chunk of chunks) { await wechat.sendMessage(chunk); }

💡 重要提醒

  • 账号安全：务必使用 微信小号 进行测试，避免主号被误判或限制。
  • 消息频率：微信对发送频率有一定限制，过于频繁可能触发风控，建议加入延时（如 200ms/条）。
  • 企业场景：若用于正式业务（如客服、通知），请优先考虑 企业微信官方 API，合规且功能更丰富。
  • 官方接口性质：目前 iLink Bot API 仍处于“实验性”阶段，策略可能随时调整，建议关注腾讯官方公告。

📌 总结

推荐路线：使用基于官方 iLink API 的工具（如 cc-weixin 或 weixin-agent-sdk），配合小号测试，并利用 SDK 自带拆分能力处理长消息。此方案兼顾了稳定性与易用性，是当前最可靠的选择。

若仅用于个人实验且能接受风险，可尝试第三方桥接，但不建议用于生产环境。
