## 一、问题与背景

### 用户问题

Orbit 当前存在三个能力缺口，通过调研确认：

1. **记忆检索只有关键词**：`memory/` 模块的搜索仅支持 BM25 关键词匹配。中文 CJK 场景下效果差，无法语义理解检索。同时记忆需要 Agent 显式调用 `record_event()` 才能写入——不能自动从执行轨迹学习。

2. **外部知识依赖**：当前 `knowledge/` 模块（SQLite + BGE + TF-IDF）只能查本地知识。外部实时信息完全缺失。调研中发现的 AnySearch 是闭源 SaaS——不能作为关键依赖。

3. **目标执行无验证**：`goal/` + `goal_judge/` 模块缺少"执行前先规划、每步验证"的显式闭环。Zerox Agent 的 Goal Mode（规划→执行→验证→汇报）是成熟参考。

### 行业对标

| 缺口 | 对标 | 差距 |
|------|------|------|
| 记忆检索 | Zerox Agent: FTS5 + BGE 向量 → RRF 混合融合 | Orbit 只有 BM25 |
| 记忆写入 | Zerox Agent: 执行完成自动从轨迹提取 | Orbit 需手动 record_event() |
| 外部搜索 | AnySearch: Agent 原生搜索（闭源） | Orbit 无外部信息源 |
| 目标闭环 | Zerox Agent: 规划→里程碑→验证→汇报 | Orbit goal/ 缺验证闭环 |

### Non-Goals（本次不做）

- 不做 UE MCP 通用桥接框架（P3）
- 不做 OpenTab 风格 TUI 仪表盘（单独迭代）
- 不做 Shell AST tree-sitter 安全分析（单独迭代）
- 不做 Toolset Registry 双层注册（单独迭代）

## 三、用户故事（P0/P1/P2）

### P0 —— 关键路径（阻塞其他功能）

| # | 用户故事 | 验收标准 |
|---|---------|---------|
| US-1 | 作为 Orbit Agent，我希望在执行代码审查时能搜索外部最新库版本和安全漏洞公告，以便给出准确的审查意见 | Agent 在审查时自动搜索；搜索结果经前置筛选（去重去噪）后注入上下文；外部源不可用时自动降级到本地 |
| US-2 | 作为 Orbit Agent，我希望记忆能自动从执行轨迹提取关键事件，而不是依赖开发者手动调用 record_event() | 每次任务执行完成后自动扫描轨迹；提取关键事件（失败/决策/异常）并自动记录到 EpisodicMemory |

### P1 —— 核心体验

| # | 用户故事 | 验收标准 |
|---|---------|---------|
| US-3 | 作为 Orbit Agent，我希望搜索记忆时能理解语义（如"上次审计为什么失败"），而不是只能匹配关键词 | 记忆检索支持 BGE 向量语义搜索；BM25 + 向量 RRF 混合排序；中文查询召回率显著优于纯 BM25 |
| US-4 | 作为 Orbit Agent，我希望拿到一个目标后能自动拆成可验证的步骤，执行后自己验证结果 | Goal 输入后自动生成规划（里程碑+成功标准）；每步执行后自动验证；验证失败时自动修正或暂停 |
| US-5 | 作为 Orbit Agent，我需要搜索外部信息时，优先用自有搜索模块，AnySearch 只作可选加速 | `knowledge/search/` 提供至少 2 个开源源（SearXNG + GitHub API）；AnySearch 仅在配置了 API Key 且可用时使用；不可用时自动降级 |

## 五、边缘情况

| 场景 | 期望行为 |
|------|---------|
| 外部搜索超时 | 3s 超时 → 自动降级到下一个源 → 最终返回本地知识图谱结果 |
| AnySearch API Key 过期/配额用尽 | 自动跳过 AnySearch → 走开源源（SearXNG/GitHub API） |
| 记忆文件超过 50KB 限制（已有截断逻辑） | 向量索引更新不因截断丢失——索引独立于文件 |
| 自动提取产生垃圾记忆 | 低效用记忆（utility < 0.1）在记忆衰减周期自动 prune |
| Goal 规划阶段 LLM 返回无效规划 | 规划 schema 校验失败 → 重试 1 次 → 仍失败时用默认线性规划 |
| 并发搜索多源结果不一致 | 按信息密度仲裁算法排序，标注来源 |
| 中文搜索 BM25 和向量分数冲突 | RRF 融合，k=60 作为平衡参数 |
| 隐私敏感数据进入搜索缓存 | 缓存仅存查询 hash + 结果 hash，不存原始内容。30 分钟 TTL |

## 七、待确认问题

1. **SearXNG 部署方式**：Docker 自托管还是用公共实例？建议：开发阶段用公共实例，生产环境提供自托管 docker-compose
