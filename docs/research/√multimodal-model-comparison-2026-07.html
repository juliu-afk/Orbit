📌 1. 执行摘要

核心结论：Orbit 不需要 claude-video 也能做视频理解。

GLM-4.1V-Thinking（2026-07-02 发布）原生支持 2 小时视频分析、GUI Agent、视觉定位——能力比 claude-video 的 ffmpeg 抽帧方案更强。DeepSeek 的 Visual Primitives Thinking（2026-04 发布）在图像理解上 token 效率碾压 Claude（90 vs 870 tokens/图）。两支国产模型在多模态上并不落后。

建议：GLM-4.1V-Thinking-Flash 免费层试水 → GLM-4.1V-Thinking-FlashX 生产。DeepSeek VL2 作为自部署备选。

🟣 3. GLM-4.1V-Thinking 旗舰

GLM-4.1V-Thinking 2026-07-02 发布

智谱最新通用视觉语言模型——当前最可用的国产多模态旗舰。

参数量~10B（9B 轻量版）
核心机制思维链推理 (CoT) + 课程采样强化学习 (RLCS)

开源HuggingFace + ModelScope 同步开源
本地部署vLLM，≥24GB VRAM（RTX 4090 / A100）
基座模型GLM-4-9B

多模态能力矩阵

能力域描述对比 Claude
📹 视频理解 原生 2 小时视频，时序分析 + 事件逻辑建模 胜出 Claude 需抽帧
🖼️ 图文理解 精准识别并综合分析图像与文本 持平
🔢 数学/科学推理 复杂题解、多步演绎、公式理解 胜出 (CoT)

📄 文档/PDF 多页文档总结、结构化提取 持平
✅ GLM-4.1V-Thinking 是 Orbit 视频理解的最佳选择：原生 2 小时视频分析意味着 Orbit agent 可以直接消化长视频——会议录像、bug 录屏、教程视频——无需 ffmpeg 预处理管线。GUI Agent 能力对 Orbit 的 UI 自动化测试也有直接价值。

🎬 8. 与 claude-video 的关系

claude-video 可复用的部分

yt-dlp 下载器覆盖 1800+ 平台，GLM/DeepSeek 都没有。Orbit 必须集成。
ffmpeg 抽帧 + 去重DeepSeek VL2（不支持视频）需要抽帧。审计截图也需要。
字幕优先策略先用原生字幕，没有才转写。节省成本。通用思路。
帧预算算法≤30s→30帧, 10min+→100帧封顶。token 控制优秀。

不需要 claude-video 的部分

Whisper 转录GLM-4.1V 原生理解视频音频，无需单独转录。
Claude 多模态调用替换为 LiteLLM → GLM/DeepSeek 路由。
Claude Code 插件机制替换为 Orbit Tool Registry + MCP。
定位：claude-video 从"主力引擎"降级为"下载器+抽帧工具"。Orbit 的核心视频理解走 GLM-4.1V 原生能力，claude-video 的 yt-dlp + ffmpeg 管线作为辅助组件。这比直接装 claude-video 插件更灵活——不绑定 Claude，不绑定 Anthropic API。
