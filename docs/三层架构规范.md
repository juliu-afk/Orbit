# Orbit 三层调用架构规范

> MCP/A2A 抽象分层 (Step 5.1/5.2 补充)

## 三层边界

```
┌─────────────────────────────────────────────┐
│  外部系统 (REST API)                         │
│  POST /api/v1/tasks                         │
└──────────────┬──────────────────────────────┘
               │
┌──────────────▼──────────────────────────────┐
│  Layer 1: 系统→Agent                         │
│  协议: asyncio.create_task() + await run()   │
│  位置: scheduler/orchestrator.py             │
│  禁止: 直接调用 MCP/A2A                      │
└──────┬───────────────────────────────────────┘
       │ await agent.run(context)
       │
┌──────▼───────────────────────────────────────┐
│  Agent 层                                    │
│                                              │
│  ┌─────────────┐  ┌──────────────────────┐  │
│  │ Layer 2      │  │ Layer 3               │  │
│  │ Agent→工具    │  │ Agent→Agent           │  │
│  │ 协议: MCP     │  │ 协议: A2A MessageBus  │  │
│  │ 位置:         │  │ 位置:                 │  │
│  │ agents/       │  │ communication/        │  │
│  │ act()→        │  │ send_a2a()→           │  │
│  │ ToolRegistry  │  │ MessageBus.request()  │  │
│  └─────────────┘  └──────────────────────┘  │
└──────────────────────────────────────────────┘
```

## 各层职责

### Layer 1: 系统→Agent (协程拉起)

| 项目 | 说明 |
|------|------|
| **位置** | `scheduler/orchestrator.py` → `_run_agent()` |
| **协议** | Python asyncio 协程调用 (非网络协议) |
| **数据** | `TaskContext` (L1-L5 五层上下文) → `AgentResult` |
| **约束** | 调度器不直接调用 MCP 工具或 MessageBus；Agent 间不通过调度器中转 |

### Layer 2: Agent→工具 (MCP)

| 项目 | 说明 |
|------|------|
| **位置** | `agents/base.py` → `act()` → `ToolRegistry.invoke()` |
| **协议** | MCP (Model Context Protocol) |
| **约束** | 权限检查/限流/版本管理由 ToolRegistry 统一管理 |

### Layer 3: Agent→Agent (A2A)

| 项目 | 说明 |
|------|------|
| **位置** | `communication/message_bus.py` → `AgentMessageBus.request()` |
| **协议** | A2A (Agent-to-Agent) via MessageBus |
| **模式** | Request-Response / Notification / Streaming / Callback |

## 静态检查规则

代码审查时逐项确认:

1. Orchestrator 中不得出现 `mcp_client` 或 `ToolRegistry.invoke()`
2. Agent 基类中不得出现 `asyncio.create_task()` 拉起其他 Agent 协程
3. Agent 间通信必须走 `MessageBus`, 禁止直接调用其他 Agent 的 `run()` 方法
4. 外部系统不直接与 Agent 交互, 必须通过 REST API → Orchestrator
