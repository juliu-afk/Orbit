# PRD：驾驶舱界面精简——聊天主区域 + 右侧摘要

| PRD (产品需求文档) |  |
| --- | --- |
| **背景** | 当前驾驶舱信息密度过高：4 指标卡片 + 3 熔断灯 + DAG 拓扑图 + Token 趋势图 + 防幻觉图 + 聊天面板 + 告警列表 + 健康面板。用户打开后面对 8 个信息方块，核心交互入口（聊天对话框）只占右栏约 38% 宽度。首次使用时认知负担高——「哪些是需要我关注的？哪里是我的主要操作入口？」 |
| **用户故事** | 作为开发者，我打开 Orbit 后看到的是一个干净的聊天对话框，右侧有简洁的摘要数字，我想深入了解某个指标时点击即可展开详情。我的注意力始终在对话上，不想被无关图表分心。 |
| **需求描述** | ① 主区域只留聊天面板（ChatPanel.vue），占满左侧 ~70% 宽度。② 右侧 ~30% 宽度为紧凑摘要面板，显示：活跃任务数、Token 消耗、防幻觉拦截数、熔断器状态（3 圆点）、告警数量（>0 时红标）、系统健康。③ 摘要面板底部 `[详情 ▸]` 按钮，点击打开 el-drawer 侧抽屉，展示完整详情：DAG 任务流 + Token 趋势图 + 防幻觉图 + 告警列表 + 健康面板。④ 无 Session 时和现在一样显示引导页。⑤ 顶部 SessionBar 不变。 |
| **范围 (Do/Don't)** | **Do：** 重排 DashboardView 布局，右侧摘要面板 + el-drawer 详情；复用现有 MetricsCard/CircuitBreakerLight/HealthPanel/TokenChart/DagCanvas/AlertList 组件。<br>**Don't：** 不新增文件、不新增 npm 依赖、不修改后端、不改 SessionBar 和 ChatPanel 内部逻辑。 |
| **边缘情况** | ① 无指标数据时摘要显示 `---`。② 告警=0 时摘要不显示告警行。③ 抽屉打开时如果 WS 推新数据，抽屉内容实时更新。④ 窗口宽度 < 768px 时右侧摘要折叠到底部。 |
| **成功标准→验收** | **SC1:** 打开界面后聊天面板占主视觉区域 → **AC1:** 聊天面板宽度 ≥ 65% 视口。<br>**SC2:** 所有原来分散的指标信息在右侧摘要可见 → **AC2:** 摘要面板显示 6 项信息（任务/Token/拦截/CB/告警/健康）。<br>**SC3:** 点击详情可查看完整信息 → **AC3:** 点击 `[详情 ▸]` 打开抽屉，内含 Token 图 + 告警列表 + 健康面板。<br>**SC4:** 不新增文件 → **AC4:** `git diff --stat` 仅 `DashboardView.vue` 一个文件。 |
| **待定决策** | 摘要面板是否需要可折叠？→ **决议：** 默认展示，窄屏（<768px）自动收到底部。 |
