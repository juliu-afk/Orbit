## 2. API 设计

### 2.1 代码图谱可视化（新增 2 个端点）

#### `GET /api/v1/codegraph/graph-data`

获取当前项目的代码图谱数据，Cytoscape elements 格式。

```
Query: project_id (str, required) — 项目 ID
Response:
{
  "code": 0,
  "data": {
    "elements": [
      // nodes
      {"data": {"id": "code:src/orbit/graph/engines/code_graph.py", "label": "code_graph.py",
                 "type": "file", "file_path": "...", "symbol_count": 15, "in_degree": 3, "out_degree": 8}},
      // edges
      {"data": {"id": "e:...", "source": "code:src/...", "target": "code:src/...",
                 "type": "imports", "weight": 1}}
    ],
    "stats": {"node_count": 150, "edge_count": 320, "built_at": "2026-07-06T10:00:00Z"}
  },
  "message": "ok"
}
```

错误码：
- `CODE_001` — 项目未构建代码索引
- `CODE_002` — 项目无 Python 文件
- `PROJ_001` — project_id 无效

#### `GET /api/v1/codegraph/graph-snapshots`（P1 时间轴）

```
Query: project_id (str)
Response:
{
  "code": 0,
  "data": {
    "snapshots": [
      {"commit_hash": "abc123", "timestamp": "2026-07-01T...", "message": "feat: add export",
       "node_count": 120, "edge_count": 250, "diff": {"added": 5, "removed": 2}}
    ]
  }
}
```

注：此端点依赖 git 仓库。无 git 时返回空数组。

### 2.2 微信集成（新增 5 个端点）

全部注册在 `API_V1_STR` 前缀下。

| 方法 | 路径 | 说明 | 响应模型 |
|------|------|------|---------|
| `POST` | `/wechat/bind/start` | 生成 bind_token + QR 码 | `{code, data: {bind_token, qrcode_data_url, expires_at}}` |
| `GET` | `/wechat/bind/status` | 查询绑定状态 | `{code, data: {status, wechat_nickname, connected_at}}` |
| `POST` | `/wechat/callback` | iLink 回调（openid + bind_token） | `{code, data: {success}}` |
| `DELETE` | `/wechat/bind` | 断开绑定 | `{code, message}` |
| `GET` | `/wechat/config` | 获取微信配置 | `{code, data: {enabled, daily_summary_time, quiet_hours}}` |
| `PUT` | `/wechat/config` | 更新微信配置 | `{code, data: {updated_fields}}` |

#### `POST /api/v1/wechat/bind/start`

```
Request: (无 body，从 Auth header 取当前用户)
Response:
{
  "code": 0,
  "data": {
    "bind_token": "d4f8...a3c2",
    "qrcode_data_url": "data:image/png;base64,iVBOR...",
    "expires_at": "2026-07-06T10:05:00Z"
  }
}
```

错误码：
- `WECHAT_001` — 已绑定且未断开
- `WECHAT_002` — cc-weixin 服务未就绪（子进程启动中/失败）
- `WECHAT_003` — QR 码生成失败

#### `POST /api/v1/wechat/callback`（iLink 服务器调用，不经过 AuthMiddleware）

```
Body:
{
  "openid": "oXXXX...",
  "bind_token": "d4f8...a3c2"
}

Response:
{"code": 0, "data": {"success": true}}

错误码：
- WECHAT_004 — bind_token 无效或已过期
- WECHAT_005 — 微信号已绑定其他用户
```

## 3. 数据模型

### 3.1 新增表：`wechat_bindings`

```sql
CREATE TABLE wechat_bindings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    orbit_user_id INTEGER NOT NULL UNIQUE,     -- Orbit 用户 ID（一个用户只能绑一个微信）
    wechat_openid TEXT NOT NULL UNIQUE,         -- 微信 OpenID（一个微信号只能绑一个用户）
    wechat_nickname TEXT,                       -- 微信昵称（展示用）
    bind_token TEXT,                            -- 当前活跃绑定令牌（NULL=已完成绑定）
    bind_token_expires_at TEXT,                 -- ISO 8601 时间戳
    status TEXT NOT NULL DEFAULT 'pending',     -- pending/active/disconnected
    connected_at TEXT,
    disconnected_at TEXT,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE UNIQUE INDEX idx_wechat_bindings_user ON wechat_bindings(orbit_user_id);
CREATE UNIQUE INDEX idx_wechat_bindings_openid ON wechat_bindings(wechat_openid);
```

### 3.2 现有表不变

| 表 | 变更 | 说明 |
|----|------|------|
| `code_nodes` | 无 | 现有字段已覆盖可视化需求 |
| `edges` | 无 | 现有 `source_node_type`/`target_node_type` 已支持跨类型边 |
| `tasks` | 无 | 微信通过现有 Task API 创建/查询任务 |

## 7. 调度器影响

**不修改调度器状态机**。微信推送通过事件订阅实现：

```python
# src/orbit/integration/wechat/outbound.py（伪代码）

async def setup_wechat_subscribers(event_bus: EventBus):
    """注册微信推送的事件订阅者。不影响现有 WS 广播。"""
    event_bus.subscribe("task:update", on_task_update)    # 完成/失败
    event_bus.subscribe("hitl:required", on_hitl_required) # 审批请求
```

事件类型新增：`hitl:required`（HITL 审批请求事件）。此事件在现有 HITL 流程中已存在内部信号，只需在 EventBus 上 publish。

## 8. 防幻觉层影响

**无影响**。两个功能均不涉及 LLM 输出。

- 代码图谱可视化：数据来自确定性 ast 解析，不经 LLM
- 微信消息路由：意图分类用简单规则匹配（前缀 `@Orbit` + 关键词），**不用 LLM 做分类**

消息路由规则（纯规则匹配，无 LLM 调用）：

| 模式 | 意图 | 处理 |
|------|------|------|
| `/^@Orbit\s+(启动\|开始\|新建\|start)\s+(.+)/` | 创建任务 | → Task API |
| `/^@Orbit\s+(查\|查询\|状态\|status)\s+#?(\d+)/` | 查询任务 | → Task API |
| `/^@Orbit\s+(同意\|批准\|approve)\s+#?(\d+)/` | 审批通过 | → HITL API |
| `/^@Orbit\s+(拒绝\|reject)\s+#?(\d+)/` | 审批拒绝 | → HITL API |
| `/^@Orbit\s+(帮助\|help\|?)/` | 帮助 | → 返回命令列表 |
| `/^@Orbit\s+(状态\|status)$/` | 全局状态 | → 列出进行中任务 |
| 其他 | 未识别 | → 返回帮助提示 |

## 9. 边界 Case 清单（硬性）

| # | 场景 | 处理方案 | 对应 PRD |
|---|------|---------|---------|
| BC-1 | 项目无 Python 文件 | API 返回 `CODE_002`，前端 empty state | E-1 |
| BC-2 | 项目 >5000 节点 | API 默认只返回顶层模块（`type='file'`），子符号按需展开 | E-3 |
| BC-3 | 图谱重新解析中 | API 返回 `built_at` 时间戳，前端 polling 等新数据 | E-5 |
| BC-4 | 无 git 仓库（时间轴） | `/graph-snapshots` 返回空数组，前端 TimeSlider disabled | E-7 |
| BC-5 | bind_token 5 分钟过期 | 前端倒计时显示，到期自动禁用→用户点"刷新" | E-2 |
| BC-6 | cc-weixin 子进程崩溃 | 自动重启（3 次），3 次均失败→API 返回 WECHAT_002 | E-4 |
| BC-7 | 微信长消息 >600 汉字 | 按 Unicode codepoint 边界切分，不在 Emoji/中文中间截断 | E-9 |
| BC-8 | 出站频率超限 | TokenBucket 队列排队，超时 30s 未发出→丢弃并记录日志 | AC-4 |
| BC-9 | 同一微信号绑两个用户 | 拒绝，返回 WECHAT_005 | E-8 |
| BC-10 | 未绑定时微信发消息 | channel.py 返回"请先在 Orbit 驾驶舱绑定微信" | AC-8 |
| BC-11 | Orbit 重启后 cc-weixin 状态 | cc-weixin 随 launcher 重启，已持久化的登录态自动恢复 | AC-7 |
| BC-12 | 窗口缩放图谱自适应 | Cytoscape canvas resize + debounce 300ms 重绘 | E-8 |
| BC-13 | 空 edges（孤立节点） | 正常渲染，不报错 | E-2 |

## 10. 风险与缓解

| # | 风险 | 严重程度 | 缓解措施 |
|---|------|---------|---------|
| R1 | **性能**：5000 节点 Cytoscape 渲染卡顿 | 中 | 默认只渲染文件级节点（~100-500），子符号按需展开。超过 2000 节点弹出性能警告。 |
| R2 | **正确性**：ast 解析与 Tree-sitter 解析结果不一致（未来切换） | 低 | 本期不改 CodeGraphEngine。统一通过 API 层抽象，切换不影响前端。 |
| R3 | **可靠性**：cc-weixin 子进程不稳定（腾讯实验性 API） | 高 | 子进程崩溃自动重启；微信不可用不影响 Orbit 任何其他功能；iLink 挂掉时 API 返回 WECHAT_002 |
| R4 | **安全**：微信消息被截获/伪造 | 中 | HTTPS；bind_token 一次性+过期；回调验证 openid 签名 |
| R5 | **可维护性**：两个独立功能耦合在一个 PR | 低 | 模块完全独立（前端不同 views，后端不同 routes），无交叉依赖。代码审查分开进行 |
| R6 | **微信封号**：频率过高触发风控 | 中 | TokenBucket 严格限流（5条/分钟，30条/小时）；强制小号 |
