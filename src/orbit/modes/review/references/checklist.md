# 代码审查清单——Orbit 特化 7 维度逐项检查

> 基于 Orbit 45 模块架构特化。移除恪现财务/会计维度。
> 对标 compose/skills/review.md 的 4 维度加权体系。

## 1. 调度器正确性（致命）
- [ ] 状态转换路径完整——所有合法转换有代码路径，非法转换被拦截
- [ ] 检查点保存/回滚策略——测试执行前保存，失败可回滚，通过后清理
- [ ] DAG 任务依赖解析——Kahn 拓扑排序正确，环形依赖抛异常
- [ ] 重试计数上限——≤3 轮，超限标记 FAILED_PERMANENT
- [ ] 离线调度稳定性——编辑检测不误触发，延迟窗口合理

## 2. 防幻觉全链路（致命）
- [ ] L1-L9 单层改动是否影响上下游判定——改一层必须审查全链路
- [ ] 测试代码也经过 L1-L9 验证——防止 Agent 生成有幻觉的测试
- [ ] L7 沙箱执行隔离——生成的代码在 Docker 内跑，宿主机不受影响
- [ ] 误报/漏报风险评估——新增验证规则需标注误报和漏报场景
- [ ] 熔断阈值合理——Token 计数器 + 延迟阈值，超限即熔断回滚

## 3. 图谱查询兼容性（严重）
- [ ] CodeGraph 查询接口向后兼容——新增查询不改已有接口签名
- [ ] 跨图谱关系完整性——代码/数据库/配置三引擎 + 元图谱 12 种关系无遗漏
- [ ] 图谱查询空结果处理——返回空列表而非抛异常
- [ ] 新增模块在 code_graph 中可被索引——Tree-sitter 解析覆盖新文件

## 4. 沙箱与安全（致命）
- [ ] 禁止 eval() / exec() / os.system() / subprocess 直连宿主机
- [ ] 硬编码密钥/密码/Token——一律从环境变量读取
- [ ] 路径遍历漏洞——文件操作限定在项目根目录内
- [ ] 生成的代码默认隔离执行——不经沙箱不进生产

## 5. Agent 角色与通信（严重）
- [ ] 新增 Agent 角色注册到 factory——不被遗忘导致 dispatch 失败
- [ ] Agent 间通信协议——消息总线模式正确，熔断传播不遗漏
- [ ] Actor 生命周期——spawn/watchdog/cleanup 路径完整
- [ ] Goal Judge 判定逻辑——fail-open，MAX_GOAL_REACT 硬上限有效

## 6. 方案偏差（严重）
- [ ] 是否按阶段 2 技术方案实现——偏离必须主动标记
- [ ] 是否有超出方案范围的改动——额外改动需说明理由
- [ ] 对应 Step 的 PRD+ADR 是否被遵守——架构决策不可静默偏离

## 7. 代码质量（一般）
- [ ] 三行相似→不抽象——重复 3 次以上才提取
- [ ] 过早抽象——只有一个调用点的泛化代码→合并回去
- [ ] 空值/边界条件——None/空列表/零值/超时 处理完整
- [ ] 新增模块在 orbit.spec hiddenimports 中注册——否则 PyInstaller 漏打包
- [ ] Ponytail 合规——无不必要抽象/stdlib 可替代/冗余包装/死代码
