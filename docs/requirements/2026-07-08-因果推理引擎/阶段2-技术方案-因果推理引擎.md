## 四、数据模型

### 4.1 因果边表（新增 SQLite 表，同 TrajectoryCollector 库）

```sql
CREATE TABLE IF NOT EXISTS causal_edges (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source_var TEXT NOT NULL,       -- agent_role / model_tier / tool_seq / principle_set
    target_var TEXT NOT NULL,       -- task_outcome / latency / tool_error_rate
    causal_strength REAL NOT NULL DEFAULT 0.0,  -- DoWhy arrow_strength()
    confidence REAL NOT NULL DEFAULT 0.0,        -- bootstrap confidence
    avg_causal_effect REAL,                      -- E[Y|do(X=x₁)] - E[Y|do(X=x₀)]
    sample_count INTEGER NOT NULL,
    last_updated REAL NOT NULL,
    UNIQUE(source_var, target_var)
);
CREATE INDEX IF NOT EXISTS idx_ce_source ON causal_edges(source_var);
CREATE INDEX IF NOT EXISTS idx_ce_target ON causal_edges(target_var);
```

### 4.3 Python 数据模型

```python
# causal/models.py
class CausalEdge(BaseModel):
    source_var: str
    target_var: str
    causal_strength: float
    confidence: float
    avg_causal_effect: float | None
    sample_count: int

class CausalGraph(BaseModel):
    """DoWhy GCM 拟合后的因果图。"""
    variables: list[str]
    edges: list[CausalEdge]
    learned_at: float
    sample_size: int
    # 底层 DoWhy GCM 对象（不序列化——运行时持有）
    _gcm_model: object | None = None  # dowhy.gcm.InvertibleStructuralCausalModel

class CauseCandidate(BaseModel):
    variable: str
    anomaly_score: float          # DoWhy gcm.attribute_anomalies() 输出
    causal_effect: float          # do-calculus 因果效应
    explanation: str = ""         # LLM 生成的人类可读解释（P1）
    counterfactual: str = ""      # 反事实建议

class RootCause(BaseModel):
    task_id: str
    causes: list[CauseCandidate]   # 异常分数降序
    top_cause: CauseCandidate
    confidence: float              # 整体置信度
    explanation_failed: bool = False  # LLM 解释是否降级
```

### 5.2 内部 Python API

```python
# causal/graph.py
class CausalModelManager:
    """DoWhy GCM 封装——因果图全生命周期管理。"""

    def __init__(self, db: sqlite3.Connection): ...

    async def build_dag(self) -> nx.DiGraph:
        """从领域知识构建初始 DAG：
        agent_role → task_outcome, tool_error_rate
        model_tier → latency, task_outcome
        tool_seq   → tool_error_rate, latency"""

    async def fit(self, min_samples: int = 50) -> CausalGraph:
        """用轨迹数据拟合 GCM——gcm.auto.assign_causal_mechanisms() + gcm.fit()"""

    async def update(self, new_trajectory_ids: list[str]) -> CausalGraph:
        """增量重拟合——仅用新增轨迹，更新边权重"""

# causal/root_cause.py
class RootCauseAnalyzer:
    def __init__(self, model: CausalModelManager, trajectory_collector): ...

    async def analyze(self, task_id: str) -> RootCause:
        """三步：① 从轨迹中提取该任务的变量值 ② gcm.attribute_anomalies(target='task_outcome')
        ③ 异常分数排序 + 反事实计算"""

# causal/recommend.py
class CausalRecommender:
    def __init__(self, model: CausalModelManager, llm_client: LLMClient | None = None): ...

    async def explain(self, root_cause: RootCause) -> str:
        """LLM 生成人类可读解释——输入异常分数 + 反事实，输出解释文本"""

    async def recommend(self, root_cause: RootCause) -> list[Recommendation]:
        """基于 do-calculus 查询生成具体改进建议"""
```

## 七、边界 case 清单

| 类别 | 场景 | 预期行为 |
|------|------|---------|
| 数据不足 | 轨迹数 <50（GCM 不可靠） | 降级为 Spearman 相关排序，返回 `confidence=low`，不调用 DoWhy |
| 数据不足 | root_cause 查询时该任务无轨迹 | 返回错误 `"任务未完成或轨迹不存在"` |
| 数据缺失 | model_tier 为空（旧轨迹） | 跳过该变量，标记 `missing_variables=["model_tier"]` |
| 算法 | GCM 拟合失败（DoWhy 异常） | 捕获异常 → 降级为相关性模式 |
| 算法 | 异常分数两个变量并列（差 <5%） | 并列返回，标记 `tie=true` |
| 算法 | 所有节点异常分数 <0.3 | 返回 `"无显著根因"` + 建议人工介入 |
| 并发 | 学习进行中同时查询根因 | 返回上一版因果图结果，标记 `stale=true` |
| 并发 | 两个学习同时触发 | 后触发者等前一个完成（SQLite 写锁） |
| LLM | LLM 解释生成失败 | 降级为纯数值报告，标记 `explanation_failed=true` |
| LLM | LLM 解释被防幻觉层拦截 | 返回标注 `[解释不可用——防幻觉层拦截]` + 数值报告 |
| 集成 | Shadow mode 因果 vs 规则结果矛盾 | 两条都记录 audit，1 周后由统计裁决 |
| 集成 | VIGIL 调用 causal 超时（>500ms） | 跳过因果，降级为规则模式，标记 `causal_timeout=true` |

## 八、风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| DoWhy GCM 在 <500 样本下拟合不稳定 | 中 | 高——异常归因不可靠 | 样本量阈值 50 + bootstrap 置信度 <0.6 标记 low_confidence |
| model_tier 数据缺失导致关键变量不可用 | 高 | 中——因果图缺一个节点 | 阶段 3 第一步补齐；旧轨迹自动 skip 该变量 |
| 新增依赖（dowhy）引入 CI 问题 | 低 | 中——CI 红 | poetry.lock 同步更新 + GitHub Actions 缓存 layer |
| 因果查询延迟 >100ms | 低 | 低 | GCM 拟合离线做（定时任务），查询仅做 attribution（<10ms） |
| LLM 解释引入幻觉到因果链 | 低 | 高——因果分析的可信度 | LLM 仅做人类可读解释——不参与因果计算；数值根因独立于 LLM |
| GEPA 因果解耦引入循环依赖 | 低 | 高——进化方向错误 | 同一代 GEPA 不使用该代自身轨迹 |
