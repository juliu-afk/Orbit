🎯 业务需求一致性分析 8 核心能力评估

1. Agent 任务调度 ★★★★★

核心调度链完整（PENDING→SCOPING→PLANNING→CODING→REVIEW→TESTING→DONE）。多角色 Agent（architect/developer/reviewer/qa/chatter/clarifier）+ Actor 子系统。扣分：SCOPING 状态引入后测试未同步。

2. 高峰避让 ★★★★★

PeakWindowManager + OffPeakScheduler + DeferredQueue 架构完整，但全量测试失败（tzdata 缺失）。前端集成完毕（peak.ts + PeakPromptDialog）。扣分：依赖管理缺失导致 CI 红灯。

3. 幻觉检测 ★★★★★

8 层检测器全部实现，Pipeline 编排完整。但生产中仅调用 validate_quick()（L1+L4+L3），L2/L5/L6/L7/L8 在生产路径中从未执行。功能存在但实际覆盖率低。

4. 上下文预构建 ★★★★★

Prebuilder + Scanner 架构设计良好，但 9 个 Context Builder 全死代码，3 个 Scanner 未集成。功能半成品。

5. 知识库 ★★★★★

知识引擎 + 向量搜索 + 本体 + MCP Server 完整。财税/会计领域知识已内置。扣分：ontology 模块仅有 accounting.py 一个领域。

6. 代码审查 ★★★★★

Ponytail 规则 + Review Service + SQLAlchemy ORM 完整。API 端点齐全。扣分：ponytail_debt 路由未注册（孤岛）。

7. 进化学习 ★★★★★

GEPA/GRPO/Distill/Inject/Scope/LlmDistill/Anchor 共 8 个模块全部实现，但 零测试覆盖。业务价值与工程投入不匹配——复杂度高但无质量保障。
