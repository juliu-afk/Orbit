## 1. 问题

Orbit 当前的 Agent 能执行任务但不能保证不漂移。9 层防幻觉（L1-L9）防止了代码层面的幻觉——但没有防止 **Agent 决策层面的错误**：目标遗忘、上下文混淆、自信地做错事。

UIUC+斯坦福+AMD 的 AgentDebug 研究证实：Agent 在行动中会逐渐偏离目标而自察不到。更危险的是——即便偏离目标，Agent 依然"自信"地输出推理，且在错误中自我循环。早期细微偏差会沿记忆、反思、规划、行动多个阶段持续积累，最终导致全局失败。

这对应了 Agent 五大能力中的 **④ 闭环容错** 和 **② 规划推理** 两个 P0 致命缺口。不补的后果：在 CIF 财务/审计场景下——Agent 做出的审计程序建议可能偏离目标且无法自察 = 执业风险。

## 2. 目标

为 Orbit Agent 运行时增加两层安全机制：

1. **元认知层 (Metacognitive Monitor)**：次级 Monitor Agent——独立于主 Agent 运行，不执行任务，只监控执行任务的 Agent 是否在偏离目标。基于预定义触发器预测失败，在不可恢复状态之前触发人工移交或自动纠正。

2. **ReflAct 反思循环**：在每个 ReAct 循环的 Observation 之后插入 Reflection 阶段——"当前状态是否仍对准原始目标？如果不是→回溯/调整"。让 Agent 成为自身的观察者。

## 3. 用户故事

### P0：必须实现

**故事 1：Agent 目标漂移被拦截**
> 作为 CIF 审计 FDE，我希望 Agent 在执行长时间的审计数据分析任务时，如果开始偏离原始目标（比如本来在查收入确认时点，逐渐跑偏到分析费用明细），能够在不可逆错误发生之前被拦截并提醒我介入，而不是默默做完然后给我一份完全偏离目标的错误结果。

**验收标准**：Monitor Agent 在检测到目标偏离时，在 3 个 Action 循环内触发告警或 HITL。误报率 < 10%。

**故事 2：Agent 每步反思对准目标**
> 作为 CIF 审计 FDE，我希望 Agent 在执行每一步之后，自动反思"这一步是否让我离目标更近了"，如果不是则自动调整或回溯。我不希望 Agent 像现在这样一口气跑完整个 DAG 才发现第一步就走错了。

**验收标准**：ReflAct 循环的 Reflection 阶段在每次 Observation 后自动执行。偏离检测准确率 > 80%。

**故事 3：重复动作被自动检测**
> 作为 Orbit 运维人员，我希望当 Agent 陷入循环（反复调用同一个工具但没有任何进展）时，Monitor 能自动检测并熔断，而不是消耗 Token 直到资源耗尽。

**验收标准**：重复动作检测在 5 个重复周期内触发。误报率 < 5%。

### P1：应该实现

**故事 4：预判式 HITL（人在回路）**
> 作为 CIF 审计 FDE，我希望在关键决策节点（如是否调整审计重要性水平），Agent 主动询问我的判断，而不是自己做决定。我不希望 Agent"代我签字"。

**验收标准**：关键决策节点（已配置的触发器）100% 触发人工移交。

**故事 5：失败模式自动分类**
> 作为 Orbit 开发人员，我希望 Agent 的失败被自动分类到标准错误类型（目标遗忘/上下文混淆/工具误用/参数错误），以便针对性优化。

**验收标准**：AgentErrorTaxonomy 分类准确率 > 75%。

## 4. 成功指标

| 指标 | 目标 | 测量方法 |
|------|------|---------|
| 目标漂移检测率 | ≥85% | 构造 50 个漂移场景的测试集，统计被 Monitor 检出的比例 |
| 误报率（Monitor） | <10% | 构造 50 个正常执行场景，统计误触发告警的比例 |
| ReflAct 偏离检测准确率 | >80% | 构造 50 个偏离场景，统计 Reflection 阶段正确判断的比例 |
| 重复动作检测 | 5 周期内 100% | 构造 20 个循环场景 |
| HITL 触发延迟 | <3 个 Action 循环 | 从检测到偏离到触发 HITL 的响应时间 |

## 5. 边缘情况

- **Monitor 自身出错**：Monitor 本身也是 Agent——它出错怎么办？需要 Monitor 自身的健康检查。Monitor 崩溃 = 主 Agent 继续运行但不被监控 = 退化到当前状态。不能更差。
- **误报过高**：Monitor 太敏感→频繁打断主 Agent→任务无法完成。需要可调节的灵敏度。
- **性能开销**：每个循环多一个 Reflection 阶段 + Monitor 监控 = Token 消耗增加 ~20-30%。需要在安全性和成本间平衡。
- **与现有防幻觉层的关系**：元认知层不是替代 L1-L9——是互补。L1-L9 防代码质量问题，元认知层防 Agent 决策质量偏差。
- **与 checkpoint/回滚的关系**：ReflAct 单步回溯粒度比分 checkpoint 更细——不需要回滚整个检查点。
