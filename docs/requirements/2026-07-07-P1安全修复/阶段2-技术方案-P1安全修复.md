## 2. 各 P1 详细设计

### P1-1: SSE Token Header

**后端** `dependencies.py`:
```python
def verify_stream_token(
    token: str | None = Header(None, alias="X-Orbit-Token"),
    token_query: str | None = Query(None, alias="token"),
) -> str:
    actual = token or token_query  # Header 优先
    if not actual or actual != settings.ORBIT_AUTH_TOKEN:
        raise HTTPException(status_code=403, detail="token 无效")
    return actual
```

**前端** `useEventSource.ts`: 原生 EventSource 不支持自定义 Header。替换为 fetch-based 轻量实现（~40 行），保持接口不变。

### P1-2: AppContainer 标记

条件导入 + 方法拆分，不实际实现 AppContainer（需 pywin32 + 大量代码）。改善代码结构的可追溯性。

### P1-3: JWT

- 新依赖: `PyJWT`
- 新配置: `JWT_SECRET` env var（默认随机）
- `AuthMiddleware`: 先匹配静态 token，再尝试验证 JWT
- `chat.py` WS: 同上双模式
- 提供 `create_jwt_token()` 工具函数

### P1-4: Layer4 强制

```python
if tool_name == "exec_command":
    if not (policy and not policy.require_sandbox):
        from orbit.sandbox.process_sandbox import ProcessSandbox
        if not ProcessSandbox().is_available():
            return False  # 沙箱不可用，拒绝
```
`is_available()` 是同步方法（仅检查 Python 解释器存在），无需 async。

### P1-5: TokenBucket Lock

```python
self._lock = asyncio.Lock()
async def allow(self, tokens: float = 1.0) -> bool:  # sync→async
    async with self._lock:
        ...
```
Breaking change: 需更新所有 `bucket.allow()` 调用方加 `await`。

### P1-6: 统一敏感文件常量

```python
# security/constants.py
SENSITIVE_FILE_GLOB: list[str] = [".env", ".env.*", "*.pem", "*.key",
    "credentials", "secrets", "id_rsa", "id_ed25519"]
SENSITIVE_FILE_NAMES: frozenset[str] = frozenset({
    ".env", ".pem", ".key", "credentials", "secrets"})
```
`guard.py` + `permission.py` 改为 import 引用。

### P1-7: dispatch 白名单

- `ToolEntry` 加 `allowed_agents: list[str] | None = None`
- `dispatch()` 新 API 路径检查: `if entry.allowed_agents and agent_name not in entry.allowed_agents: return "权限拒绝..."`
