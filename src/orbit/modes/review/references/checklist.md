# 代码审查清单——6 维度逐项检查

## 1. 安全（致命）
- [ ] SQL 注入 / XSS / 命令注入
- [ ] eval() / exec() / 动态代码执行
- [ ] 硬编码密钥/密码/Token
- [ ] 路径遍历漏洞

## 2. 财务（致命）
- [ ] 金额使用 Decimal（禁止 float/double）
- [ ] 借贷成对（debit == credit）
- [ ] 金额变量带单位后缀（_yuan / _cents）

## 3. 会计规则（致命）
- [ ] 恒等式成立（assets == liabilities + equity）
- [ ] 科目余额方向正确
- [ ] 过账不可逆

## 4. 方案偏差（严重）
- [ ] 是否按阶段2方案实现
- [ ] 是否有超出方案范围的改动

## 5. 测试覆盖（严重）
- [ ] 新代码是否有对应测试
- [ ] 核心模块是否有正+异常用例

## 6. 代码质量（一般）
- [ ] 三行相似→不抽象
- [ ] 过早抽象
- [ ] 空值/边界条件
