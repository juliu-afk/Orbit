## 发现

### P2-1: ws/router.py — 未使用导入

`src/orbit/ws/router.py:9,12`：`import asyncio` 和 `import structlog` / `logger` 均未在本文件使用。

- `asyncio`：broadcaster 协程在 `main.py` 中用 `asyncio.create_task` 启动，router.py 不需要
- `structlog`/`logger`：文件中无日志调用

**修复**：删除 `import asyncio`、`import structlog`、`logger = structlog.get_logger()`。

### P2-2: ws/manager.py — disconnect 日志计数错误

`src/orbit/ws/manager.py:40`：
```python
for task_id in list(self._rooms):
    self._rooms[task_id].discard(ws)
    if not self._rooms[task_id]:
        del self._rooms[task_id]
logger.info("ws_disconnect", rooms_cleaned=len(list(self._rooms)))
```

循环结束后 `self._rooms` 已被清空（该 WS 从所有房间移除，空房间被删除）。`len(list(self._rooms))` 始终报告 0。

**修复**：循环前保存计数：
```python
initial_rooms = len(self._rooms)
# ... 循环 ...
logger.info("ws_disconnect", rooms_cleaned=initial_rooms - len(self._rooms))
```

### P2-3: DashboardView.vue — computed 重复导入

`src/frontend/src/views/DashboardView.vue:50,61`：
```typescript
import { onMounted, onUnmounted, ref } from 'vue'   // line 50
...
import { computed } from 'vue'                        // line 61
```

Vue 响应式 API 应从同一 import 导入。

**修复**：合并到 line 50：
```typescript
import { computed, onMounted, onUnmounted, ref } from 'vue'
```

### P2-4: DagCanvas.vue — as any 类型断言

`src/frontend/src/components/dag/DagCanvas.vue:27,34`：vis-network 9.x 类型定义不完整（`Network` 构造函数、`setData` 方法无 TS 重载），被迫使用 `as any`。

**评估**：可接受。vis-network 9.1.x 类型定义由 DefinitelyTyped 维护，滞后于实际 API。运行时行为正确（已验证构建通过）。后续 vis-network 升级到 v10+ 后移除 `as any`。
