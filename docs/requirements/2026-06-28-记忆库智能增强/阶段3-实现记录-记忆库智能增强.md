## 三、偏差说明

| 方案 | 偏差 | 理由 |
|------|------|------|
| Phase 1: 迁移 SQL | 未写独立迁移文件（用 FTS5 文档存储自动适配） | Memory 条目采用文件存储（非 SQL 表），frontmatter 解析器已有向后兼容性 |
| Phase 3: Graph-RAG 存储 | `_import_edges` 用内存字典而非 SQL 邻接表 | ADR #5 决策改为内存字典——代码图谱 `incremental_update` 频繁全量重建，SQL upsert 开销大。内存字典够快，重启后 AST 重新解析自然恢复。 |
| Phase 3: HyDE 生成 | `_generate_hyde_questions` 当前为占位符（返回空字符串）；LLM 调用留待 async wrapper 就绪 | 方法签名+接口已就位，测试覆盖占位符行为。真实 LLM 调用需要 non-blocking async wrapper，避免阻塞 append_to_file |
| Phase 3: 方法命名 | `find_callers_cross_file()` → `find_definitions_cross_file()`（R1 修复） | 语义修正——实际功能是"查其他文件中的同名定义"，不是"查调用者" |
