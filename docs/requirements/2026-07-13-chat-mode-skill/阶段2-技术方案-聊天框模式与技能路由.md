## 4. 数据模型

### 4.1 ChatSkill（`src/orbit/skills/models.py`）

```python
class ChatMode(StrEnum):
    MANUAL = "Manual"
    EDIT_AUTO = "Edit Automatically"
    PLAN = "Plan"
    AUTO = "Auto Mode"

class SkillTriggerType(StrEnum):
    SLASH = "slash"       # /xxx 精确匹配
    NATURAL = "natural"   # 自然语言模糊匹配
    CHAIN = "chain"       # 作为编排链的一部分

class ChatSkill(BaseModel):
    name: str                          # "code-review"
    description: str                   # 一句话描述——用于自动匹配
    triggers: list[str]                # 触发词 ["审查", "review", "检查代码"]
    phase: str                         # plan|implement|review|verify|merge|chat
    tools: list[str]                   # 允许的工具列表
    agent_role: str                    # 执行 Agent 角色
    body: str                          # Markdown body——Skill prompt
    is_chat_skill: bool = True         # True=聊天框可调用
    is_chainable: bool = False         # True=可作为编排链一环
    version: str = "1.0.0"            # 语义化版本
    path: str = ""                     # SKILL.md 文件路径

class SkillVersion(BaseModel):
    """版本历史条目"""
    version: str
    changed_at: str                    # ISO datetime
    changed_by: str                    # "user" | "system"
    diff_summary: str                  # 简短变更说明
    file_hash: str                     # 用于回滚定位

class SkillMatchResult(BaseModel):
    skill: ChatSkill
    confidence: float                  # 0.0-1.0
    trigger_type: SkillTriggerType
    matched_by: str                    # 匹配到的触发词/理由
```

### 4.3 执行上下文（`src/orbit/core/context.py`）

```python
import threading
from dataclasses import dataclass, field

@dataclass
class ExecutionContext:
    chat_mode: ChatMode = ChatMode.AUTO
    session_id: str = ""
    # Edit Automatically: 会话级已确认的工具集合
    confirmed_tools: set[str] = field(default_factory=set)

_ctx = threading.local()

def get_context() -> ExecutionContext:
    if not hasattr(_ctx, 'value'):
        _ctx.value = ExecutionContext()
    return _ctx.value

def set_context(**kwargs):
    ctx = get_context()
    for k, v in kwargs.items():
        if hasattr(ctx, k):
            setattr(ctx, k, v)
```

## 6. 数据流

### 6.1 模式门禁（四级）

```
ToolRegistry.dispatch(tool_name, args, agent_name)
  │
  ├── 1. 已有 PermissionEngine 检查
  │
  ├── 2. ChatMode 门禁（NEW）
  │     ctx = get_execution_context()
  │     switch ctx.chat_mode:
  │       case MANUAL:
  │         → 弹确认（读+写都弹）
  │         → 等待用户确认 → 执行 / 拒绝 → 返回结果
  │       case EDIT_AUTO:
  │         → is_write=False → 自动放行
  │         → is_write=True 且 tool_name in ctx.confirmed_tools → 放行
  │         → is_write=True 且首次 → 弹确认 + "本会话记住"选项
  │           → 记住 → 加入 ctx.confirmed_tools → 放行
  │           → 不记住 → 仅本次放行
  │       case PLAN:
  │         → is_write=True → 拒绝，返回"Plan 模式不允许写操作"
  │         → is_write=False → 放行
  │       case AUTO:
  │         → 放行
  │
  └── 3. 执行工具
```

### 6.2 Skill 路由

```
用户输入 "帮我审查一下最近改的代码"
  │
  ▼
ChatterAgent.execute()  ← prompt 含 skill 匹配规则
  │
  ├── intent="chat" → 直接回复
  ├── intent="programming" → ClarifierAgent（不改）
  ├── intent="skill", skill_name="code-review", confidence=0.85
  │     → confidence ≥ 0.7 → 直接触发
  │     → 告知用户 "🔍 启动代码审查技能..."
  │     → SkillRegistry.load("code-review")
  │     → AgentFactory.create(agent_role) → 注入 skill.body 作 system_prompt
  │     → 流式执行 → 返回结果
  │
  └── intent="chain", skills=["plan","implement","review"]
        → SkillRegistry.build_chain(skills)
        → ComposeOrchestrator.run_skill_chain(skills, user_text)
        → 流式输出每个 phase 的进度
```

### 6.4 热更新流程

```
用户编辑 SKILL.md（GUI 或手动）
  → 文件系统 watcher 检测到变更
    → watchdog/fsnotify 事件
```

### 7.2 模式门禁（ToolRegistry.dispatch() 扩展）

在 `dispatch()` 第 314 行 PermissionEngine 之后插入：

```python
# Phase X: ChatMode 四级门禁
from orbit.core.context import get_context
ctx = get_context()
tool_entry = self._entries.get(name)
is_write = tool_entry.is_write if tool_entry else True  # 默认写（安全）

if ctx.chat_mode == ChatMode.PLAN:
    if is_write:
        return "Plan 模式——不允许写操作。方案已生成，切换到其他模式后可执行。"

if ctx.chat_mode == ChatMode.MANUAL:
    # 每次弹确认——返回确认请求（由上层 Agent 处理）
    if not self._request_confirm(name, args, ctx.session_id):
        return f"Manual 模式——{name} 等待用户确认中..."

if ctx.chat_mode == ChatMode.EDIT_AUTO:
    if is_write and name not in ctx.confirmed_tools:
        # 首次弹确认
        confirmed, remember = self._request_confirm(name, args, ctx.session_id, remember=True)
        if not confirmed:
            return f"Edit Automatically 模式——{name} 被用户取消。"
        if remember:
            ctx.confirmed_tools.add(name)
```

### 7.7 热更新（File Watcher）

使用 Python `watchdog` 库（Orbit 已有依赖）：

```python
class SkillWatcher:
    """监听 skills/ 目录变化 → 触发 SkillRegistry.reload()"""

    def __init__(self, registry: SkillRegistry, watch_dirs: list[Path]):
        self._registry = registry
        self._observer = Observer()
        for d in watch_dirs:
            self._observer.schedule(
                SkillFileHandler(registry),
                str(d), recursive=False
            )

    def start(self): self._observer.start()
    def stop(self): self._observer.stop()

class SkillFileHandler(FileSystemEventHandler):
    def on_modified(self, event):
        if event.src_path.endswith('.md'):
            name = _extract_skill_name(event.src_path)
            self._registry.reload(name)
```

debounce 500ms 防止编辑器连续写触发多次重载。

## 8. 边界 case 清单

| 场景 | 预期行为 | 实现 |
|------|---------|------|
| WebSocket 断连时切换模式 | UI 即时响应，重连后同步 | 本地 ref 更新；下次 send 携带 |
| Skill registry 为空 | 斜杠不弹出，自然语言跳过 | `list_all()` 为空 → filtered=[] → 不弹窗 |
| 用户输入以 `/` 开头但不是命令 | 当作普通文本走 Phase 1 意图分类 | Phase 0 匹配不到 → 继续 Phase 1 |
| 同 prompt 匹配多个 Skill（置信度接近） | 前端弹候选列表（最多 3 个） | `match_by_text()` 返回排序结果 |
| Manual 模式弹确认太频繁 | 同一 tool+args 5 秒内不重复弹 | ExecutionContext 加 `last_confirm_time` |
| Plan 模式用户说"帮我写文件" | 生成文件内容但不写入 | ToolRegistry 拦截；Agent 告知"以上为方案" |
| Skill 链中某步失败 | 暂停，展示失败节点+选项 | ComposeOrchestrator 已有错误处理 |
| 热更新时 SKILL.md 格式错误 | 保留旧版本，log error | `reload()` 内 try/except → 校验失败保留旧 |
| 回滚 Skill 到不存在版本 | 返回 404 | Version 查询 + 404 |
| mode 字段缺失（旧客户端） | 默认 Auto Mode | `get_context()` 默认值 |
| 创建 Skill 时 name 冲突 | 返回 409 Conflict | `create()` 检查重复 |
| 前端补全列表请求失败 | 降级为 FALLBACK_CMDS | InputBox 已有 fallback 机制 |

## 9. 风险与缓解

| 风险 | 可能性 | 影响 | 缓解 |
|------|--------|------|------|
| mode 状态在 Agent 执行中丢失 | 中 | 中 | thread-local context，Agent 执行前刷新 |
| 工具 is_write 标记遗漏 | 中 | 高 | 安全默认——未标记=is_write=True |
| Skill 匹配误判 | 低 | 低 | 误判走现有 chat/programming，功能不降级 |
| 热更新文件写入竞态 | 低 | 中 | debounce 500ms + 写锁 |
| ComposeOrchestrator 新入口破坏已有管道 | 低 | 高 | 不改 run_spec()，新增独立方法 run_skill_chain() |
| WebSocket 确认消息增加延迟 | 低 | 低 | Manual 模式才走确认；Edit 模式仅首次 |
