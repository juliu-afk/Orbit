# 阶段1 PRD：EvolveR LLM 语义蒸馏 + GRPO 策略优化

> 日期：2026-07-05 | 分支：feat/phase-e-evolution | 基础：PR#202合并后master

## 1. 问题

Phase C 的 DistillationEngine 只做规则蒸馏（轨迹→原则的去重+评分），没有 LLM 驱动的语义蒸馏和 GRPO 强化学习。当前能力⑤覆盖率 ~65%。

EvolveR 原论文的四阶段循环（在线交互→离线自蒸馏→经验库维护→策略进化），我们只覆盖了②③。缺①结构化轨迹的 LLM 语义提炼和④基于 GRPO 的策略优化。

## 2. 目标

- E1: LLM 语义蒸馏引擎——批量轨迹→LLM提炼共同模式→高质量策略原则
- E2: GRPO 策略评分——基于下游任务成功率动态调整原则效用分数
- E3: 策略自动注入——高效用原则自动注入 Agent system prompt

## 3. 验收标准

| # | 标准 |
|---|------|
| AC1 | LLM 蒸馏从 ≥3 条同类轨迹中提炼出 ≥1 条可复用原则 |
| AC2 | GRPO 评分在 10 次任务后原则效用排序与人工评估一致性 >70% |
| AC3 | 高效用原则（>0.7）自动注入 system prompt，Agent 引用率 >30% |
| AC4 | 蒸馏不引入危险原则（ANCHOR 护栏拦截率 100%） |
| AC5 | 蒸馏后能力⑤覆盖率 ≥80% |

## 4. Non-Goals

- 不引入外部 RL 训练框架（GRPO 用规则评分模拟）
- 不修改 LLM 权重
- 不替代 Phase C 的规则蒸馏（两者互补）
