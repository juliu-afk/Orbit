## 审查发现的缺陷

### 🔴 Critical: SkillWatcher handler 不继承 FileSystemEventHandler

- **位置**: [watcher.py](src/orbit/skills/watcher.py) `_SkillFileHandler`
- **问题**: 原版用 `__getattr__` 返回 `_handle`，但 watchdog 调度 `on_modified(event)` 时 `__getattr__` 返回的是方法引用而非调用结果，导致事件无法正确处理
- **修复**: 改用 `__new__` 动态创建 `FileSystemEventHandler` 匿名子类，正确覆盖 `on_modified` / `on_created`
- **Commit**: a804437

### 🟡 Major: AgentRole 转换在 try 块外

- **位置**: [chat.py:658](src/orbit/api/routes/chat.py) `_execute_skill()`
- **问题**: `AgentRole(skill.agent_role)` 在 try/except 外执行，SKILL.md 中无效的 `agent_role` 值会导致未捕获的 ValueError
- **修复**: 移入 try 块内，无效值降级为 `AgentRole.DEVELOPER`
- **Commit**: a804437

### 🟢 Minor: 新文件缺单元测试

- `src/orbit/skills/registry.py` — SkillRegistry 核心方法无测试
- `src/orbit/skills/models.py` — 模型验证无测试
- `src/orbit/core/context.py` — thread-local 上下文无测试
- `src/orbit/skills/watcher.py` — watcher 无测试
- 建议: 阶段 4 至少补 registry.find_by_slash + match_by_text 单元测试
