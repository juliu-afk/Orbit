## 二、`knowledge/search/` —— 自建搜索模块（新建）

### 2.1 架构

```
Agent 查询
  → IntentRouter (LLM 判断意图 → 选数据源组合)
  → 并行查询各数据源
    ├── WebSource (SearXNG API)
    ├── CodeSource (GitHub Search API)
    ├── AcademicSource (Semantic Scholar API)
    └── AnySearchAdapter (可选, 有 API Key 时)
  → Ranker (同源衰减 + 信息密度仲裁 + RRF 融合)
  → Extractor (正文提取 → Markdown)
  → SearchCache (TTL 缓存命中/未命中)
  → 返回结构化结果给 Agent
```

### 2.2 文件结构

```
src/orbit/knowledge/search/
├── __init__.py           # 导出 search(query) -> SearchResult
├── intent_router.py      # IntentRouter: LLM 判断意图 → 选源
├── sources/
│   ├── __init__.py
│   ├── base.py           # SearchSource 抽象基类
│   ├── web.py            # WebSource: SearXNG API 封装
│   ├── code.py           # CodeSource: GitHub Search API
│   ├── academic.py       # AcademicSource: Semantic Scholar
│   └── anysearch.py      # AnySearchAdapter: POST api.anysearch.com/v1/search
├── ranker.py             # 同源衰减 + 信息密度仲裁
├── extractor.py          # readability-lxml 正文提取 + html2text
└── cache.py              # SQLite TTL 缓存
```

### 2.3 核心接口

```python
# sources/base.py — 数据源抽象
class SearchSource(ABC):
    name: str
    priority: int  # 越小越优先
    
    @abstractmethod
    async def search(self, query: str, max_results: int = 10) -> list[RawSearchItem]:
        ...
    
    async def health_check(self) -> bool:  # 返回 False 时自动跳过
        ...

@dataclass
class RawSearchItem:
    title: str
    url: str
    snippet: str
    source_name: str
    relevance_score: float = 0.0

# ranker.py — 排序
def dedup_by_url(items: list[RawSearchItem]) -> list[RawSearchItem]: ...
def same_source_attenuation(items: list[RawSearchItem], decay: float = 0.7) -> list[RawSearchItem]: ...
def info_density_arbitration(items: list[RawSearchItem]) -> list[RawSearchItem]: ...
def hybrid_rank(items: list[RawSearchItem], query: str) -> list[SearchResult]: ...

# cache.py — 缓存
class SearchCache:
    def get(self, query_hash: str) -> list[SearchResult] | None: ...
    def set(self, query_hash: str, results: list[SearchResult], ttl: int = 1800): ...
    def hit_rate(self) -> float: ...
```

### 2.4 数据流

```
search("Python async best practices", max_results=10)
  → IntentRouter.route() → ["code", "web"]  # 代码类查询，优先 GitHub + 通用搜索
  → parallel [
      CodeSource.search() → 5 results,
      WebSource.search() → 8 results,
    ]
  → dedup_by_url() → 11 unique
  → same_source_attenuation() → 调整同域权重
  → info_density_arbitration() → 优先信息量大的
  → hybrid_rank() → 语义+时效混合排序 → Top 10
  → SearchCache.set(query_hash, results)
  → return SearchResult[]
```

### 2.5 降级链

```
AnySearch (有 Key 且健康) → WebSource (SearXNG) → CodeSource → AcademicSource
         ↓ 失败                        ↓ 失败              ↓ 失败
    下一个源                      下一个源            返回本地 knowledge/ 结果
```

每个源有独立超时（3s）和熔断器（连续 3 次失败 → 自动跳过该源 5 分钟）。

**修改文件**：`memory/store.py` — `search()` 方法

```python
def search(self, query: MemorySearchQuery) -> list[MemorySearchResult]:
    # 现有 BM25 分支
    bm25_results = self._search_bm25(query)  # 提取现有逻辑
    
    # 新增向量分支
    if self._vector_index:
        vector_results = self._search_vector(query)
        # RRF 融合: score = 1/(k + rank_bm25) + 1/(k + rank_vector)
        return self._rrf_fusion(bm25_results, vector_results, k=60)
    
    return bm25_results  # 向量索引未就绪时回退
```

**RRF 算法**（~15 行）：
```python
def _rrf_fusion(ranked_a, ranked_b, k=60):
    scores = {}
    for rank, item in enumerate(ranked_a):
        scores[item.id] = scores.get(item.id, 0) + 1/(k + rank + 1)
    for rank, item in enumerate(ranked_b):
        scores[item.id] = scores.get(item.id, 0) + 1/(k + rank + 1)
    return sorted(scores.items(), key=lambda x: -x[1])
```

## 六、风险点

| 风险 | 影响 | 缓解 |
|------|------|------|
| BGE 模型加载增加内存 | 已经加载一次（knowledge/），复用不增加 | 确认 `BGEEmbedder` 是单例 |
| SearXNG 公共实例不稳定 | 外部搜索降级到本地 | 公共实例仅开发用，生产强制自托管 docker-compose |
| 自动提取产生噪音记忆 | 记忆质量下降 | 只提取 CRITICAL/HIGH 重要性事件 + 定期 prune |
| Goal 规划 LLM 输出格式不稳定 | 规划解析失败 | Schema 校验 + 重试 1 次 + 默认线性规划 fallback |
