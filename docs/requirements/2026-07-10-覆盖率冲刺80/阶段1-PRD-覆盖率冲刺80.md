# 阶段1-PRD-覆盖率冲刺80.md

> 基于 Orbit 生产部署就绪检查报告 B1 阻断项

## 背景

当前覆盖率 **69.92%**（25,257 语句，6,825 缺失），CI 门禁 69%（pyproject.toml 写 95% 但未执行）。生产就绪门槛 ≥80%，差距约 1,774 条语句（≈7%）。

## 用户故事

| 优先级 | 故事 |
|--------|------|
| P0 | 作为运维，我需要覆盖率 ≥80%，以便生产部署门禁通过 |
| P1 | 作为开发者，我需要零覆盖模块至少达到基本覆盖（≥60%），以便新增代码有安全网 |
| P2 | 作为 QA，我需要核心调度/防幻觉模块分支覆盖提升，以便降低生产 Agent 异常概率 |

## 验收标准

1. `pytest tests/unit/ tests/integration/ --cov=src/orbit --cov-fail-under=80` 通过
2. 零覆盖模块（0%）从 10 个减至 0 个（每个 ≥60%）
3. 核心模块（scheduler/、hallucination/、gateway/）覆盖率不下降
4. 不引入新的测试失败（现有 1 条失败 B2 另行修复）

## 成功指标

- 综合覆盖率：69.92% → ≥80%
- 零覆盖模块：10 → 0
- 测试数量：增量 ≥50 条

## Non-Goals

- 不修复现有 1 条测试失败（B2 单独处理）
- 不提升 pyproject.toml 门禁 95%（留到后续迭代）
- 不改变业务逻辑（仅加测试）
- 不变异测试/混沌测试覆盖（P2 迭代）

## 边缘情况

- 零覆盖模块可能依赖 Docker/GPU（gui_tools, video_tools）→ Mock 隔离
- 5 层权限引擎测试需要伪造安全上下文
- LiteLLM 调用需 mock，不连真实 API

## 策略

**先 Tier 1（零覆盖→基本覆盖），再 Tier 2 精选（高 ROI 核心模块），Tier 3 留到后续。**

预估：Tier 1 全覆盖 ≈ +2.0%，Tier 2 选 6 个模块 ≈ +3.5%，分支覆盖自然增长 ≈ +4.5% → 合计 ≈ 80%。
