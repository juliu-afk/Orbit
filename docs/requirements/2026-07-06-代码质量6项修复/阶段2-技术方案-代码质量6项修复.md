### AC4: P0 核心链路异常分层

**影响文件**：3 个文件，~15 处 `except Exception`

**分层策略**：
```
可恢复异常（已知、预期内）       → WARNING + 触发降级/重试
  - SandboxExecutionError        (task_runner.py)
  - LLMTimeoutError              (gateway/client.py)
  - CheckpointConflictError      (checkpoint/manager.py)
  - asyncio.TimeoutError         (task_runner.py)

不可恢复异常（代码 bug/基础设施） → CRITICAL + exc_info + raise
  - 其他所有 Exception
```

**实现方式**——不改 try 块逻辑，只改 except 块：
```python
# 改前
except Exception as e:
    logger.error("xxx_failed", error=str(e))

# 改后
except (SandboxExecutionError, asyncio.TimeoutError) as e:
    logger.warning("xxx_recoverable", error=str(e))
except Exception as e:
    logger.critical("xxx_unexpected", exc_info=True)
    raise
```

**具体改动点**：

| 文件 | 行数* | 可恢复异常类型 | 备注 |
|------|:--:|------|------|
| `task_runner.py` | ~8处 | `SandboxExecutionError`, `asyncio.TimeoutError`, `CheckpointConflictError` | `_agent_cycle`, `_run_agent`, `resume`, `_continue_from` |
| `gateway/client.py` | ~5处 | `LLMTimeoutError` | LLM 调用、重试、降级路径 |
| `checkpoint/manager.py` | ~2处 | `CheckpointConflictError` | save/load 并发冲突 |

*行数 = 被修改的 except 块数量，非新代码行数

**边界 case**：
| 场景 | 预期 |
|------|------|
| 之前被吞的未知异常现在 raise | 上层 Scheduler 有全局 try/except → 任务标记 FAILED + audit 记录 |
| 之前靠 except Exception 兜底工作的场景 | 仅 P0 15 处收紧，P1/P2 层不变——不影响 |
| SandboxExecutionError 不存在于某些路径 | 仅在有沙箱调用的代码路径加，无沙箱场景不加 |
| raise 后任务中断率上升 | 短期可能，但每次中断有完整 audit trail，暴露的是真实 bug |

### AC5: type:ignore 消除

**影响文件**：~12 个文件，~16 处改动

**A 组——字符串→枚举转换（7 处）**：

| 文件:行 | 改前 | 改后 |

| `task_runner.py:550` | `role=role` | `role=AgentRole(role)` |
| `offpeak_scheduler.py:600` | `status=row["status"]` | `status=DeferredStatus(row["status"])` |
| `observability.py:183` | `outcome=outcome` | `outcome=typing.cast(Literal["success","failure"], outcome)` |
| `db_graph.py:160` | `return node.meta.get(...)` | `result = node.meta.get(...); assert isinstance(result, list); return result` |

**C 组——assert 窄化（4 处）**：

| 文件 | 改前 | 改后 |
|------|------|------|
| `checkpoint/manager.py:162` | `return await self.redis.get(key)` | `assert self.redis is not None; return await self.redis.get(key)` |

**边界 case**：
| 场景 | 预期 |
|------|------|
| AgentRole(role) 传非法值 | StrEnum 构造失败 → ValueError，属于代码 bug 应暴露 |
