## 4. 数据流

```
L5: func_code → parse @formal decorator → Z3 Solver.check()
    ├── unsat → passed=True  (无反例=正确)
    ├── sat   → passed=False (有反例) + counterexample
    └── timeout → passed=True + warning (不阻断)

L6: code + openapi_path → prance解析 spec → AST提取路由
    → 比对 request_model/response_model/status_codes
    → 不一致 → passed=False + differences

L7: code → 构造 pytest 风格断言代码 → Sandbox.run()
    → stdout 含测试结果 → 解析 PASSED/FAILED
    → 失败 → passed=False

L8: baseline_dir 文件 → SHA256 指纹
    → 对比当前文件 hash → 不一致 → diff
    → auto_fix? → 恢复基线 → 告警
```

## 5. 边缘 case 清单

| 场景 | 预期行为 |
|------|---------|
| @formal 缺失 | 跳过，z3_status="skipped" |
| Z3 30s 超时 | passed=True, z3_status="timeout", warning |
| OpenAPI spec 不存在 | passed=False, errors=["spec not found: ..."] |
| 沙箱不可用 | L7 跳过, passed=True + warning |
| 基线目录为空 | L8 返回 [] |
| 配置规范化后相同 | 无漂移（SHA256 基于规范化文本） |
| Prod 禁止 auto_fix | auto_fixed=False, 仅告警 |
| Z3 表达式解析失败 | passed=False, errors=["parse error: ..."] |

## 6. 风险与缓解

| # | 风险 | 缓解 |
|---|------|------|
| R1 | Z3 对复杂表达式指数爆炸 | 30s 硬超时 + 仅标记函数启用（人工筛选） |
| R2 | L6 OpenAPI spec 大文件解析慢 | 使用 prance 异步解析 + 缓存 |
| R3 | L8 自动修复覆盖用户有意变更 | Prod 禁用 auto_fix，仅告警 |
| R4 | L7 沙箱执行测试代码可能无限循环 | Sandbox 已有 timeout 机制（30s） |
