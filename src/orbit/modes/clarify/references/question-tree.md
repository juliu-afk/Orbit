# 决策树模板——ClarifierAgent 提问顺序

## 分支遍历规则

按以下顺序深度优先遍历每个决策分支：

### 分支 1: 目标 (Goal)
1. 用户要解决什么核心问题？
2. 成功的定义是什么？（可观测、可度量）
3. 这个问题为什么现在需要解决？
→ 分支 1 所有子问题 resolved → 进入分支 2

### 分支 2: 范围 (Scope)
4. 需要改动哪些模块/文件？
5. 明确不做什么？
6. 边界条件是什么？（空数据/极限值/并发）
→ 分支 2 resolved → 进入分支 3

### 分支 3: 验收 (Acceptance)
7. 怎么验证做完了？（具体步骤，可观测动词）
8. 是否有性能/安全/兼容性要求？
→ 分支 3 resolved → PRD ready

## 按需分支（对话中出现相关线索才开启）

- 数据/状态/并发 → 开启"边界条件"子分支
- 性能/安全/兼容 → 开启"非功能约束"子分支
- bug 类需求 → 开启"复现条件"子分支
